# VIC03 결과 분석 (260301)

## 1. 실험 설정

VIC01/02에서 발견된 PD 게인 버그를 수정한 첫 번째 실험.

수정 내용:
- _init_pd_gains_from_mjcf() 메서드 추가: MJCF XML의 joint user 속성에서 kp/kd를 직접 파싱
- dof_prop['stffness'] (= 0) 대신 MJCF의 실제 값 사용
- 파싱 결과: 69 DOFs, kp 범위 [50, 500], kd 범위 [1.0, 5.0]

학습 설정:
- task_reward_w: 0.5, disc_reward_w: 0.5
- power_reward: True, power_coefficient: 0.00005
- reward_specs: k_pos=100, k_rot=10, k_vel=0.1, k_ang_vel=0.1, w_pos=0.5, w_rot=0.3, w_vel=0.1, w_ang_vel=0.1
- Stage 1 (CCF=0 고정), 15,000 에폭

## 2. 결과

| 지표 | VIC02 (토크=0) | VIC03 (토크 정상) |
| :--- | :--- | :--- |
| 학습 말미 Avg Reward | ~235 | ~200 |
| 학습 말미 Avg Steps | ~85 | ~70 |
| 평가 Avg Reward | 222.3 | 254.0 |
| 평가 Avg Steps | 69.6 (2.3초) | 79.1 (2.6초) |
| 평가 Max Steps | - | 223 (7.4초) |

학습 초반(1~100 에폭)에서는 음수 reward와 5~11 steps의 매우 짧은 에피소드가 관찰되었다. 이는 랜덤 정책 + 실제 토크 = 즉시 넘어짐으로, 토크가 정상적으로 작동한다는 증거이다. VIC01/02에서는 토크=0이라 오히려 초반 reward가 높았던 것과 대조적이다.

## 3. 시각화 관찰

- 잘 걷는 에이전트가 있음: steps 223 (7.4초)짜리도 나타남. reference motion을 거의 끝까지 추종
- 하지만 여전히 발을 떼지 못하고 서서 떨리기만 하는 에이전트도 존재
- VIC02와의 결정적 차이: VIC02는 관성으로 미끄러지듯 움직였지만, VIC03는 실제로 발을 들어 올리며 보행하는 에이전트가 있음
- 걷는 에이전트와 서있는 에이전트의 성능 분산이 큼

## 4. 분석

(1) PD 게인 수정 효과 확인됨
- 실제 토크가 작동하면서 보행이 가능해짐
- 최대 7.4초까지 보행 추종 성공 사례 존재

(2) 수렴 부족 가능성
- 15k 에폭으로는 VIC의 확장된 액션 공간(q_target + CCF)에서 충분히 수렴하지 못했을 수 있음
- V4 (isaac_pd, CCF 없음)는 같은 15k 에폭에서 av reward 461.8, av steps 143.5를 달성했음
- VIC03의 254.0 / 79.1은 V4의 55% / 55% 수준

(3) 서서 떨리는 에이전트 원인
- CCF=0 고정으로 모든 관절이 기본 PD gain으로 고정되어 있어 "뻣뻣하게 서있기"가 local optimum이 될 수 있음
- power_reward가 움직임을 억제하는 방향으로 작용할 가능성 (서있으면 velocity가 낮아 penalty가 작고, 걸으면 penalty가 큼)

## 5. 결론

PD 게인 수정으로 실제 보행이 가능해졌으나, V4 대비 성능 격차가 크다. 일부 에이전트는 잘 걷지만 다수가 서서 떨리는 문제가 남아있다. reward weight 조정이 필요하다.
