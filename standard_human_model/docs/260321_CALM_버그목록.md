## BUG-01 — l_opt 단위 버그 (심각도: 🔴 Critical)

**위치**: `config/healthy_baseline.yaml` 전체 근육

**문제**:
```yaml
# 현재 (잘못됨)
quadriceps_L:
  l_opt: 1.0   # 무차원으로 되어 있음

# 코드 내부 계산 (muscle_model.py)
l_norm = muscle_length / l_opt
# muscle_length는 moment_arm.py에서 미터 단위로 계산 (~0.3m)
# → l_norm ≈ 0.3 / 1.0 = 0.3  ← 최적 길이의 30% 수준
```

**영향**: `f_FL(l_norm)` 은 `l_norm=1.0` 에서 피크. `l_norm=0.3` 이면 Gaussian에서 거의 0 → **능동 근육력이 최대치의 ~30%에 불과**. 전체 보행 토크 심각하게 과소평가.

**수정**:
```yaml
# Rajagopal 2016 Table 2 기준값으로 교체
quadriceps_L:   l_opt: 0.099   # m
hamstrings_L:   l_opt: 0.109   # m
soleus_L:       l_opt: 0.044   # m
gastrocnemius_L: l_opt: 0.051  # m
tibialis_ant_L: l_opt: 0.068   # m
hip_flexors_L:  l_opt: 0.100   # m
gluteus_max_L:  l_opt: 0.142   # m
# 우측도 동일값 사용
```

**수락 기준**: mid-stance에서 soleus `l_norm` ∈ [0.9, 1.05]

## BUG-02 — R 행렬 각도 의존성 없음 (심각도: 🟠 High)

**위치**: `core/moment_arm.py`

**문제**:
```python
# 현재: R이 상수 텐서 (num_muscles, num_dofs)
self.R = torch.tensor(...)  # 초기화 시 1회만 설정

# 실제: moment arm은 관절 각도에 따라 최대 ±30% 변화
# 예) hamstrings: 무릎 0° → r=3.5cm, 무릎 90° → r=5.5cm
```

**영향**: 관절 각도 극단에서 토크 예측 오차 최대 30%. 특히 무릎 굴곡이 큰 계단 보행, 앉기 동작에서 오차 심화.

**수정 방향**: polynomial fitting 적용
```python
# r_i(q_j) = a0 + a1*q_j + a2*q_j^2
# 계수는 OpenSim gait2392에서 추출
def _compute_R(self, dof_pos: Tensor) -> Tensor: ...
```

**참고**: 평지 보행 범위(±30°) 내에서는 오차 ~10% 수준이므로 BUG-01 수정 후 차순위.

## BUG-03 — Elastic Tendon 미구현 (심각도: 🟠 High)

**위치**: `core/muscle_model.py`

**문제**: `compute_force()`는 rigid tendon 가정. 아킬레스건(soleus, gastrocnemius)은 보행 에너지의 35~40%를 저장·반환하는 핵심 스프링이지만 현재 완전히 무시됨.

**영향**: terminal stance push-off 파워 과소평가. Winter 2009 기준 발목 A2 파워 피크가 3.5 W/kg인데 시뮬에서 재현 불가.

**수정 방향**: `soleus`, `gastrocnemius`에만 elastic tendon 추가
```python
def compute_tendon_force(self, l_tendon): ...   # Zajac 1989
def compute_force_elastic_tendon(self, activation, l_mtu, v_mtu): ...
# Newton-Raphson 반복으로 l_ce 수렴 (최대 10회)
```

**수락 기준**: 보행 중 아킬레스건 에너지 저장 30~40 J/cycle (Ker 1987)

## BUG-04 — Stretch Reflex 지연 없음 (심각도: 🟡 Medium)

**위치**: `core/reflex_controller.py`

**문제**:
```python
# 현재: 근육 신장 감지 → 즉각 반사 활성화 (delay=0)
a_stretch = stretch_gain * max(0, v_muscle - threshold)

# 실제: 척수 반사弓 지연 25~60ms
# delay_steps = round(0.040 / dt)  # 40ms @ dt=1/120s → 5 steps
```

**영향**: 경직(spastic) 환자 시뮬에서 클로누스 발현 타이밍 부정확. 정상 보행에서는 영향 미미하나 L3 검증(클로누스 onset 테스트)에서 오류 발생.

**수정 방향**:
```python
# 근육 속도 히스토리 버퍼 추가
self.velocity_buffer = deque(maxlen=delay_steps)
v_delayed = self.velocity_buffer[0]  # delay_steps 전 값 사용
```
