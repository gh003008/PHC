# 웨어러블 로봇 통합을 위한 관측 가능 근골격 관절 역학 시뮬레이션

---

## 1. 문제 정의

### 1.1 연구 배경

다양한 환자군(뇌졸중 편마비, 파킨슨병, 근력약화, 경직 등)을 보조하는 웨어러블 로봇을 개발하기 위해서는, 시뮬레이션 내 인간 모델이 해당 환자의 병리적 신경근 특성을 충실히 모사할 수 있어야 한다. 그러나 Isaac Gym과 같은 skeleton 기반 시뮬레이션은 관절 토크를 직접 지정하는 구조여서 경직, 강직, 속도 의존적 근력 변화 등 근육에서 기인하는 병리적 역학을 체계적으로 표현할 수 없다. 반면 MyoSuite와 같은 근골격 시뮬레이션은 개별 근육을 모델링하여 높은 생리학적 충실도를 달성하지만, 수백 개에 달하는 파라미터 중 대부분이 비침습적으로 측정 불가능하여 실제 환자의 특성을 모델에 반영하는 Real-to-Sim 전환이 구조적으로 어렵다. 결국, 환자의 병적 특성을 충분히 모사하면서도 임상 측정을 통한 파라미터 규명이 가능한 수준의 근육 역학을 갖춘 시뮬레이션 프레임워크가 필요하다.

**예시 — 뇌졸중 환자 A:** 환측 하지에 경직이 있고 근력이 정상의 40%로 감소. 임상에서 등속성 근력검사 결과, 진자 검사에서의 경직도, 수동 관절가동범위 검사 데이터를 얻을 수 있다. 이 측정값들을 시뮬레이션 내 인간 모델에 바로 반영할 수 있다면 — 즉, 이 환자의 경직도와 근력 수준을 가진 가상 인간이 시뮬레이션에서 걷는다면 — 이 환자에게 어떤 웨어러블 로봇 세팅이 적합한지, 어떤 보조 전략이 효과적인지를 시뮬레이션에서 사전 탐색할 수 있다.

**예시 — 파킨슨병 환자 B:** 근력 자체는 크게 줄지 않았지만, 관절의 강직이 심하여 느릴 때도 빠를 때도 저항이 있다. 임상에서 이것은 상호억제 장애로 인한 과도한 동시수축으로 설명되는데, 이런 특성을 시뮬레이션에 반영하려면 관절 토크를 직접 지정하는 것으로는 부족하다. 근육의 동시수축 메커니즘이 구현되어야 한다.

**예시 — 노인 환자 C:** 근감소증으로 전반적 근력이 떨어지고, 특히 빠른 움직임에서의 힘 생성이 더 크게 감소하며, 근육의 활성화 속도도 느려져 있다. 이것은 Type II 섬유 소실에 의한 힘-속도 관계의 변화와 활성화 시간상수의 변화로 설명되는데, 단순히 최대 토크를 줄이는 것만으로는 이 특성을 정밀하게 재현할 수 없다.

이러한 사례들이 보여주는 것은, **Sim-to-Real gap을 줄이기 위해 실제 환자의 임상 측정값을 시뮬레이션에 직접 반영하는 Real-to-Sim 파이프라인이 필요**하고, 이를 위해서는 근육의 역학적 특성들이 관절 수준에서 적절히 모사되어야 한다는 점이다.

### 1.2 환자별 특성을 모사하기 위해 필요한 근육 역학 특성

위의 환자 사례에서 필요한 근육 역학을 정리하면 다음과 같다. 이것들은 단순히 생리학적 완성도를 위해 필요한 것이 아니라, **환자별 특성을 시뮬레이션에 정밀하게 반영하기 위해 구조적으로 필요한 특성들**이다.

- **힘 생성 특성 (F_max, f_FL, f_FV):** 환자 C와 같이 단순 근력 감소를 넘어 속도 의존적 힘 생성 능력 변화를 표현하려면 force-length, force-velocity 관계가 필요하다. 등속성 근력검사 데이터로 직접 규명 가능.
- **동시수축에 의한 가변 임피던스:** 환자 B의 강직은 순 토크와 독립적으로 관절 강성이 변하는 현상이다. 이는 주동근/길항근의 Hill 모델 비선형성에서 자연스럽게 창발되며, 토크 직접 지정으로는 표현 불가능.
- **반사 역학 (stretch reflex, 상호억제):** 환자 A의 경직은 신전 반사 과항진으로, 환자 B의 강직은 상호억제 장애로 설명된다. 반사 이득과 역치 파라미터를 진자 검사로 규명 가능.
- **다관절근 커플링 (R(q) 행렬):** 하지직거상 검사에서 무릎 각도에 따라 고관절 굴곡 저항이 달라지는 현상은 bi-articular 근육 커플링에 의한 것으로, 관절별 독립 모델로는 재현 불가.
- **수동 역학 및 활성화 동역학:** 수동 ROM 검사에서의 토크-각도 관계, 급속 수축 시의 토크 상승/하강 시간이 각각 passive stiffness와 activation time constant로 직접 대응.

### 1.3 기존 접근의 한계

**근골격 시뮬레이션 (MyoSuite 등):** 위의 근육 역학을 모두 포함하지만, 개별 근육 수준의 수백 개 파라미터 중 대부분이 비침습적으로 측정 불가능하다. 예컨대 병원에서 환자 A의 등속성 검사 결과를 얻었다 해도, 이 데이터로 MyoSuite 모델의 개별 근육 tendon slack length나 optimal fiber length를 역으로 구하는 것은 근육 Redundancy 문제로 인해 구조적으로 불가능하다.

**Isaac Gym 토크 모델:** GPU 병렬화와 로봇 통합에는 최적이지만, 관절 토크를 직접 지정하는 구조여서 위에서 언급한 근육 역학 특성들을 표현할 방법이 없다. 환자 A의 경직, 환자 B의 강직, 환자 C의 속도 의존적 근력 감소를 체계적으로 파라미터화 할 수 없다.

> **결론:** 임상 측정으로부터 파라미터를 규명할 수 있는 수준의 근육 역학을, Isaac Gym 환경 내에서 관절 수준으로 구현하는 프레임워크가 필요하다.

**네 가지 요구사항:**

1. **근육 역학의 관절 수준 반영:** 동시수축에 의한 가변 임피던스, 다관절근에 의한 관절 간 커플링, 경직/강직 등 병리적 역학 포함
2. **파라미터 관측가능성:** 모든 모델 파라미터가 임상에서 비침습적으로 측정 가능하여 Real-to-Sim 전환이 실현 가능해야 함
3. **로봇 통합 환경:** GPU 병렬 강화학습 및 로봇 모델과의 공동 시뮬레이션을 지원하는 Isaac Gym 환경에서 동작
4. **환자군 범용성:** 동일한 모델 구조에서 파라미터 변경만으로 근력약화, 뇌졸중, 파킨슨, 경직 등 다양한 환자군에 대응 가능

### 1.4 연구 질문

> 근골격 역학을 관절 수준 표현으로 얼마나 축소할 수 있으면서도, 가변 임피던스, 관절 간 커플링, 병리적 역학을 유지하고, 동시에 모든 파라미터가 비침습적 임상 측정으로 규명 가능한 상태를 유지할 수 있는가?

### 1.5 가설

근육군 수준(15~20개 기능적 그룹)의 가상 근육 레이어를, 제어기와 물리엔진 사이에 **수학적 모델 형태의 근육 동역학 layer**로 삽입하면, 표준 임상 생체역학 측정(등속성 근력검사, 수동 ROM 검사, 교란 검사, 진자 검사)으로 파라미터를 규명할 수 있는 수준에서 근골격 역학의 핵심 관절 수준 발현을 재현할 수 있다. 이를 통해 병원에서 측정한 환자 데이터를 시뮬레이션에 직접 전환하는 Real-to-Sim 파이프라인을 구축할 수 있다.

### 1.6 반영되는 근육 역학 특성 전체 목록

| 분류 | 설명 |
|------|------|
| **힘 생성 특성** | 근육 길이에 따른 힘 생성 능력 변화(force-length), 수축 속도에 따른 힘 변화(force-velocity), 근육군별 최대 힘 생성 상한(F_max) |
| **수동 역학** | 근육 신장 시 활성화와 무관하게 발생하는 탄성 저항력(passive force-length), 근육 내부 점성 감쇠, 인대/관절낭에 의한 ROM 끝단 저항(soft limits) |
| **시간 역학** | 신경 명령에서 근육 활성화까지의 시간 지연(activation dynamics). 활성화가 비활성화보다 빠른 비대칭성 포함 |
| **구조적 커플링** | R(q) 행렬을 통한 bi-articular/multi-articular 커플링. 한 관절의 움직임이 다른 관절의 능동/수동 저항에 영향 |
| **자발적 창발 특성** | 동시수축 시 Hill 모델 비선형성에서 자연 발생하는 가변 관절 임피던스. 상태 의존적 비선형 토크 생성 |
| **반사 역학** | 신전 반사(stretch reflex), 상호억제(reciprocal inhibition), GTO 반사, 척수 경유 시간 지연(25~40ms) |

### 1.7 단일 프레임워크의 환자군 범용성

본 프레임워크의 파라미터들은 인간 신경근 시스템의 서로 다른 병리적 기전에 독립적으로 대응하도록 설계되어 있다.

| 병리 | 매핑 파라미터 |
|------|-------------|
| 근력약화 | F_max ↓, v_max ↓ |
| 경직 | g_reflex ↑, v_threshold ↓ |
| 파킨슨 강직 | g_reciprocal ↓ |
| 노화 | F_max ↓, τ_act ↑ |

각 병리가 서로 다른 파라미터 부분집합에 매핑되므로, 복합 질환(예: 경직 + 근력약화 동시 존재)도 해당 파라미터들을 조합하여 표현할 수 있다. 동일한 보조기기가 서로 다른 환자군에서 어떻게 작용하는지를 **파라미터만 바꾸며 비교 평가** 가능하고, Real-to-Sim 파이프라인을 통해 특정 환자의 개인화된 파라미터를 구축하여 맞춤화된 보조기기 세팅이나 재활 전략을 시뮬레이션에서 사전 탐색할 수 있다.

## 2. 프레임워크 구조

### 2.1 계층 구조

본 프레임워크는 세 계층으로 엄격히 분리된다:

1. **골격계 (URDF):** 뼈의 질량/관성, 관절 축, 하드 ROM, 컨택 메시. 초기화 시 1회 로드. 연부조직 역학 미포함. URDF로 구현.
2. **근육 및 하위 신경계 (환경 스크립트):** `pre_physics_step()`에서 PyTorch GPU 텐서 연산으로 근육 역학 및 연부조직 역학 전체를 구현. **프레임워크의 핵심.**
3. **상위 신경계 (상위 제어기):** 강화학습, CPG, 임피던스 제어기, 임상 데이터 재생 등. 환경 레이어 수정 없이 교체 가능.

### 2.2 Timestep별 계산 흐름

매 timestep마다 환경 레이어에서 다음 7단계가 순차적으로 실행된다. 모든 연산은 GPU에서 병렬 환경에 걸쳐 batch 처리된다.

```
┌─────────────────────────────────────────────────────────────┐
│                    상위 제어기 (RL / CPG / ...)              │
│                    → 운동 명령 u 출력                        │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  [근육 동역학 레이어 — pre_physics_step() 내 GPU 텐서 연산]  │
│                                                             │
│  단계 1: 현재 상태 읽기 (q, dq ← Isaac Gym solver)          │
│  단계 2: 척수 반사 레이어 (stretch reflex, 상호억제)         │
│  단계 3: 활성화 동역학 (1차 ODE: τ_act / τ_deact)           │
│  단계 4: R(q) → 근육 운동학 (l_m, v_m 계산)                 │
│  단계 5: Hill 모델 힘 생성 (F_active + F_passive)            │
│  단계 6: 관절 토크 매핑 (τ = R(q)ᵀ × F_total + soft limit)  │
│  단계 7: 토크 인가 (set_dof_actuation_force_tensor())        │
│                                                             │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              Isaac Gym Physics Solver                        │
│              → 다음 q, dq 계산 → 단계 1로 피드백              │
└─────────────────────────────────────────────────────────────┘
```

**각 단계 상세:**

| 단계 | 내용 | 비고 |
|------|------|------|
| **1** | Isaac Gym solver로부터 관절 위치 `q`와 속도 `dq`를 읽기 | |
| **2** | 근육 신장 속도를 속도 역치(`v_threshold`)와 비교, 초과 시 시간 지연(25~40ms)을 거쳐 반사 활성화 신호 생성. 하행 명령에 반사 성분 합산 | |
| **3** | 1차 ODE로 신경 신호와 기계적 힘 사이의 지연 모델링. `τ_act`(10~50ms), `τ_deact`(40~200ms) | |
| **4** | 모멘트 암 행렬 `R(q)`가 관절 상태를 근육 길이(`l_m`)와 속도(`v_m`)로 변환. 이관절근의 R 행에 여러 관절의 0이 아닌 값 → 관절 간 커플링 발생 | |
| **5** | `F_active = a × F_max × f_FL(l/l_opt) × f_FV(v/v_max)`, `F_passive = F_max × f_PE(l/l_opt)` (l_opt 이상 신장 시만 발생) | |
| **6** | `τ_muscle = R(q)ᵀ × F_total`. 이관절근 커플링 토크 자동 생성. 인대/관절낭 소프트 리밋 토크 추가 | |
| **7** | `τ_total`을 `set_dof_actuation_force_tensor()`로 Isaac Gym solver에 전달. Solver가 다음 q, dq를 계산하여 단계 1로 피드백 | |

### 2.3 입출력 인터페이스

- **입력:** 상위 제어기의 운동 명령 `u` + solver로부터의 현재 관절 상태 (`q`, `dq`)
- **출력:** 관절 토크 `τ`

> **핵심 차이:** 동일한 `u`라도 현재 `q`, `dq`에 따라 다른 `τ`가 생성된다. 이것이 단순 토크 모델과의 근본적 차이이다.

### 2.4 동시수축에 의한 가변 임피던스

관절 임피던스는 Hill 모델의 비선형성에서 자연스럽게 **창발**한다. 주동근과 길항근이 동시 활성화되면, 순 토크는 거의 0이지만 외부 교란은 한쪽을 신장(`f_FV` 신장성 증강으로 힘 증가)시키고 다른 쪽을 단축(힘 감소)시켜 자동으로 복원 토크를 생성한다. 동시수축 수준이 높을수록 이 복원력의 기울기(강성)가 증가한다.

### 2.5 Hill 모델의 3요소 구조와 건(tendon) 처리

전통적 Hill-type musculotendon 모델의 세 요소:

| 요소 | 설명 | 수식 |
|------|------|------|
| **CE** (Contractile Element) | 활성화에 의해 힘을 생성하는 수축 요소 | `a × F_max × f_FL × f_FV` |
| **PE** (Parallel Elastic) | 근섬유와 병렬 수동 탄성 요소, l_opt 이상 신장 시 저항력 | `F_PE(l/l_opt)` |
| **SE** (Series Elastic) | 근섬유와 직렬 건, 자체 force-length 관계 보유 | `F_SE(l_tendon)` |

CE와 PE는 병렬이므로 힘 합산: `F_fiber = F_CE + F_PE`
직렬 평형 조건: `F_SE = F_fiber × cos(α)`

#### 본 프레임워크의 선택: Rigid Tendon 가정

CE와 PE는 **완전히 구현**. SE(건)는 **rigid tendon 가정** 적용하여 초기 모델에서 생략. 세 가지 이유:

1. **계산 효율성:** Elastic tendon → 매 timestep `F_fiber = F_tendon` 평형 조건의 비선형 방정식 반복 풀이(Newton-Raphson) 필요 → GPU SIMD 병렬 처리와 맞지 않아 성능 병목. Rigid tendon → 근섬유 길이가 대수적으로 바로 결정.
2. **파라미터 관측가능성:** SE → `k_tendon`, `l_slack_tendon` 파라미터 추가인데, 임상 비침습 측정 극히 어려움. Rigid tendon → 건의 compliance 효과가 PE의 `k_PE`에 흡수. 실제 임상에서도 PE와 SE의 수동 저항 분리 측정 불가 → 하나의 `f_PE`로 합쳐 표현하는 것이 관측가능성 원칙에 부합.
3. **Lumping 용이성:** 같은 근육군 내 개별 근육들의 건 길이/강성이 서로 달라 하나의 대표 SE 파라미터로 합치기 어려움. CE와 PE는 F_max 합산, k_PE 가중합 등으로 lumping이 자연스러움.

#### 아킬레스건: SE 구현 권장 (예외)

아킬레스건은 예외적으로 **elastic tendon**을 구현할 것을 권장한다:

- 인체에서 가장 길고 강한 건(약 15cm)
- 보행 stance phase에서 체중에 의해 늘어났다가 push-off에서 탄성 반동으로 에너지 방출
- 보행 에너지 효율의 **35% 이상**이 이 메커니즘에서 기인
- 대상: **가자미근**과 **비복근** (둘 다 아킬레스건 공유)의 2개 근육군
- 건의 force-length 관계: `F_SE = k_tendon × ((l_tendon - l_slack) / l_slack)²`

나머지 하지 근육군에서는 SE가 큰 의미 없음:
- 건의 compliance가 기능적으로 의미를 가지려면 건이 충분히 길어 탄성 에너지 저장량이 유의미해야 하고, 보행 중 stretch-shortening cycle을 겪어야 함
- 대퇴사두근(슬개건): 건이 짧음
- 햄스트링: 제동 역할이 주
- 장요근/둔근 등: 건이 짧거나 근섬유 대비 건 비율이 작음

#### 계산 비용: 병목 없음

2개 근육군만의 elastic tendon은 계산 병목이 되지 않는다. 평형 방정식 `F_fiber = F_tendon`은 1차원 비선형 방정식으로, 함수가 단조이므로 Newton-Raphson이 **3~5회 반복**이면 수렴. 전체 20개 근육군 중 18개는 rigid tendon으로 배치 텐서 연산 한 번이면 끝나고, 2개만 짧은 반복 루프를 돌리므로 추가 비용 미미. (모든 근육(80+개)에 elastic tendon → 80 × 5 = 400회/timestep 반복 → GPU 병목과는 완전히 다른 상황)

> **요약:** R(q) 행렬은 순수 기하학적 관계(모멘트 암)만 담당하고, 근건 단위의 역학적 특성(CE, PE, SE)은 힘 생성 단계에서 처리. 대부분 근육군은 rigid tendon 하 CE+PE가 전부, 아킬레스건(가자미근/비복근)에만 elastic tendon(SE)을 추가하여 push-off 효율 재현.

### 2.6 Level A/B 전략: 충실도와 규명 용이성의 유연한 선택

#### Level A: 근육군 단위 모델링

- 상위 제어기의 action → 근육군별 neural command (15~20개)
- 각 근육군은 `R(q)`에서 독립적인 행 보유
- bi-articular/multi-articular 근육 → 해당 행에 여러 관절의 0이 아닌 moment arm
- 관절 간 커플링이 `R(q)` 구조에서 자연스럽게 발생
- 특정 근육군의 F_max 변경 → 해당 근육이 걸치는 모든 관절의 능동력/수동력/임피던스가 일관성 있게 변화
- EMG 데이터가 있을 때 활성화 검증 가능

#### Level B: 관절별 주동근/길항근 모델링

- 상위 제어기의 action → 관절별 agonist/antagonist command (관절당 2개)
- 각 관절에 가상의 단관절 주동근/길항근 쌍
- action space가 작아 (6관절 양측 = 24개) 제어기 설계 간단
- 모든 파라미터가 관절 수준 측정만으로 규명 가능 → **EMG 불필요**
- 단점: bi-articular 커플링은 명시적 커플링 항 `C_ij(q)`로 표현해야 함, 특정 근육의 선택적 병리(예: 비복근만 경직) 표현 어려움

#### 하이브리드 접근 (권장)

가장 강력한 접근은 Level A + B 하이브리드:

- **Level A (명시적 근육군):** 주요 bi-articular 근육군 — 햄스트링, 비복근, 대퇴직근
- **Level B (관절별 agonist/antagonist):** 나머지 단관절 근육군

→ action space = bi-articular 3~5개 + 관절별 10~12개 = **~15개**
→ Level A의 커플링 충실도 + Level B의 규명 용이성 동시 확보

#### 단계적 연구 전략

1. **Level B로 초기 구축** → 프레임워크 전체 검증
2. **주요 bi-articular 근육군을 Level A로 추가** → 점진적 충실도 향상

각 단계가 독립적으로 동작하는 완성된 모델이며, Level B만으로도 보행 제어기 학습, 환자 파라미터화, Real-to-Sim이 모두 가능. Level A 요소 추가 시 충실도 향상되지만, 빠져 있다고 프레임워크가 무너지지 않음.

### 2.7 R(q) 행렬 구성: SMPL 23관절 모델 대응

SMPL 기반 골격 모델: 23개 관절 × XYZ 3축 회전 = **69 DOF**

- `R(q)` 행렬: 열 = 69개 DOF, 행 = 근육군 수
- 근육의 line of action을 각 관절 회전축에 대해 분해 → 축별로 다른 moment arm
- 대부분의 원소가 0인 매우 **희소(sparse) 행렬**

> **주의:** SMPL의 각 관절 local frame XYZ가 해부학적 축(flexion/extension, abduction/adduction, internal/external rotation) 중 어느 것에 대응하는지를 관절마다 확인하여 정확히 매칭해야 한다.

| 파라미터 | 설명 | 대표값 | 규명 방법 |
|----------|------|--------|----------|
| `F_max` | 최대 등척성 힘 (N) | 사두근 ~4000, 전경골근 ~600 | 등속성 근력검사 peak torque ÷ moment arm |
| `l_opt` | 최적 근섬유 길이 (f_FL 정점) | 근육별 (m) | 문헌 (OpenSim), 체격 스케일링 |

| `v_max` | 최대 단축 속도 (l_opt/s) | 속근 ~10, 지근 ~4 | 다속도 등속성 검사 |
| `A_f` | f_FV 단축성 곡률 파라미터 | ~0.25 | 토크-속도 곡선 피팅 |
| `f_M_len` | 신장성 힘 비율 (신장성/등척성) | ~1.5 | 신장성 등속성 검사 |
| `k_PE` | 수동 탄성 강성 계수 | ~4.0 | 수동 ROM 검사 토크-각도 곡선 |
| `ε₀` | F_passive = F_max 도달 최대 변형률 | ~0.6 | 수동 ROM 검사 토크-각도 곡선 |

#### 모멘트 암 행렬 R(q)

`(n_근육군 × n_관절)` 행렬. 각 원소는 해당 근육군이 해당 관절에 대해 가지는 모멘트 암(m). 이관절근은 행당 2개 이상의 0이 아닌 값. OpenSim 모델(Rajagopal 2016)에서 추출, 체격 스케일링으로 개인화 가능.

#### 활성화 동역학 파라미터

| 파라미터 | 설명 | 대표값 | 규명 방법 |
|----------|------|--------|----------|
| `τ_act` | 활성화 시간상수 (근육 켜는 시간) | 10~50 ms | 급속 수축 시 토크 상승 시간 |
| `τ_deact` | 비활성화 시간상수 (τ_act의 2~4배) | 40~200 ms | 수축 중단 후 토크 감소 시간 |

#### 반사 레이어 파라미터

| 파라미터 | 설명 | 대표값 | 규명 방법 |
|----------|------|--------|----------|
| `g_reflex` | 신전 반사 이득 | 정상 ~0.2 / 경직 0.6~1.0+ | 진자 검사, 교란 검사 |
| `v_threshold` | 반사 속도 역치 | 정상: 보통 / 경직: 매우 낮음 | 진자 검사 응답 분석 |
| `t_delay` | 반사 시간 지연 (근방추→척수→근육) | 25~40 ms | H-반사 잠복기 |
| `g_reciprocal` | 상호억제 이득 | 정상: 보통 / 파킨슨: 낮음 | 동시수축 비율 측정 |

#### 인대/관절낭 파라미터 (관절별)

| 파라미터 | 설명 | 대표값 | 규명 방법 |
|----------|------|--------|----------|
| `k_ligament` | 인대 강성 계수 (Nm/rad) | 관절별 상이 | 수동 ROM 끝단 저항 |
| `q_onset` | 소프트 리밋 시작 각도 | ROM 경계 근처 | 수동 ROM 검사 |
| `α` | 지수적 가파름: `τ = -k · exp(α · (q - q_onset))` | ~10~20 /rad | ROM 끝단 곡선 형태 |

## 4. Real-to-Sim 파라미터 규명

### 4.1 원칙: 순차적 분리 규명

각 실험 조건이 특정 파라미터 부분집합만 활성화시키므로, 단계별로 분리하여 규명할 수 있다. 이는 전체 근골격 모델(MyoSuite)에서는 근육 Redundancy로 인해 **구조적으로 불가능한 접근**이다.

### 4.2 규명 파이프라인

#### 1단계: 수동 역학 (활성화 = 0)

- **실험:** 수동 ROM 검사. 완전 이완 상태(EMG 무신호 확인)에서 각 관절을 느리고 일정한 속도로 움직이며 각도-저항 토크 기록
- **대상:** `k_PE`, `ε₀`, `d_m`, `k_ligament`, `q_onset`, `α`
- **방법:** 활성화=0이므로 능동 힘 파라미터가 제거되어 수동 파라미터만 분리 규명

#### 2단계: 최대 힘 용량 (활성화 = 1.0)

- **실험:** 등속성 근력검사. 여러 각속도(60°/s, 120°/s, 180°/s, 240°/s)에서 최대 수의 수축
- **대상:** `F_max`, `v_max`, `A_f`, `f_M_len`
- **방법:** 1단계에서 고정된 수동 토크를 차감 후, 능동력 모델을 토크-속도 데이터에 피팅

#### 3단계: 활성화 동역학

- **실험:** 급속 수의 수축/이완 과제
- **대상:** `τ_act`, `τ_deact`
- **방법:** F_max와 f_FL/f_FV가 이미 알려져 있으므로, 토크 상승/하강 시간 프로파일에서 시간상수만 분리

#### 4단계: 반사 파라미터

- **실험:** 진자 검사 또는 교란 검사
- **대상:** `g_reflex`, `v_threshold`, `g_reciprocal`
- **방법:** 모든 근육 파라미터가 결정된 상태에서, 시뮬레이션 진자 응답을 실제 응답에 매칭

#### 5단계: 통합 검증

- **실험:** 보행 분석 (motion capture + force plate)
- 1~4단계의 파라미터로 모델을 구동하여 예측 관절 토크와 시공간적 보행 파라미터를 실측값과 비교

## 5. 환자별 파라미터화

### 5.1 원칙

모든 환자군은 **동일한 모델 구조** 내에서 **파라미터 수정만으로** 표현된다.

### 5.2 예시: 허벅지 근력약화

**메커니즘:** 대퇴사두근/햄스트링의 힘 생성 능력 저하. 근감소증, 폐용성 위축, 운동 신경 손실 등이 원인.

| 파라미터 | 정상값 | 약화값 | 근거 |
|----------|--------|--------|------|
| `F_max` (사두근) | ~4000 N | 1600~2400 N (40~60%) | 등속성 검사로 측정 |
| `v_max` | ~10 l_opt/s | ~6~7 l_opt/s | Type II 섬유 소실로 고속 힘 감소 |
| `τ_act` | ~15 ms | ~25 ms | 느려진 신경 동원 |

**예상 거동:** 입각기 loading response에서 사두근 토크 부족으로 구부정 보행 또는 보상적 무릎 과신전. 유각기에서 v_max 감소로 활보장 단축, 분속수 감소. 동시수축 임피던스도 비례적으로 감소하여 외부 교란 취약성 증가.

### 5.3 예시: 경직 (속도 의존적 과긴장)

**메커니즘:** 상위 운동 신경 병변으로 신전 반사에 대한 하행 억제가 제거되어 반사가 과항진. 만성화 시 근육 조직 변화(섬유화) 추가.

| 파라미터 | 정상값 | 경직값 | 근거 |
|----------|--------|--------|------|
| `g_reflex` | ~0.2 | 0.6~1.0+ | 하행 억제 소실 |
| `v_threshold` | 보통 | 매우 낮음 (×0.3~0.5) | 느린 신장에서도 반사 유발 |
| `g_reciprocal` | 보통 | 감소 (×0.3~0.5) | 상호억제 장애 |
| `k_PE` | ~4.0 | 6.0~12.0 (×1.5~3) | 만성 조직 변화 |

**예상 거동:** 유각기 무릎 굴곡 시 사두근 신전 반사가 과도하게 발동 → 비자발적 수축이 무릎 굴곡 방해 → **stiff-knee gait**. 느린 움직임에는 저항 적고 빠른 움직임에는 강한 저항(속도 의존성). k_PE 증가는 속도 무관한 기저 저항을 추가.

### 5.4 추가 환자군

| 질환 | 주요 파라미터 변경 | 관찰 현상 |
|------|-------------------|----------|
| 파킨슨병 | `g_reciprocal` ↓↓ (강직), 기저 동시수축 ↑, 진전 잡음 추가 | 납관 강직 |
| 편마비 | 이환측: `F_max` ↓, `τ_act` ↑, `g_reflex` ↑ | 비대칭 보행 |
| 노인 | `F_max` ↓ (전반), `v_max` ↓, `τ_act` ↑ | 느린 보행, 짧은 활보장 |

## 6. 프레임워크 검증

### 6.1 수준 1: 구성요소 검증

각 하위 모델을 알려진 생체역학적 관계에 대해 독립적으로 검증:

- **힘-길이 관계:** 다양한 관절 각도에서 등척성 힘이 예상되는 종형 곡선과 일치
- **힘-속도 관계:** 단축성 토크 감소, 신장성 토크 > 등척성 일치 확인
- **이관절근 커플링:** SLR 검사에서 무릎 신전/굴곡에 따른 고관절 굴곡 저항 변화 재현

### 6.2 수준 2: 현상 검증

- **동시수축 임피던스:** 동일 교란에 대해 동시수축 수준에 비례하여 관절 강성이 단조 증가 확인
- **반사 응답:** 진자 검사에서 `g_reflex`에 따라 체계적으로 달라지는 진동 감쇠 패턴 재현

### 6.3 수준 3: 규명 파이프라인 교차 검증

- **훈련-검증 분리:** 부분 실험 조건으로 규명한 파라미터가 보류 조건에서의 토크를 허용 오차 내로 예측
- **교차 양식 예측:** 정적 검사에서 규명한 파라미터가 동적 거동을 재피팅 없이 예측

### 6.4 수준 4: 통합 보행 검증

- **건강인 기준선:** 제어기가 생성한 보행의 시공간적 파라미터가 문헌 표준 범위 내
- **병리적 보행 재현:** 경직 파라미터 → stiff-knee gait, 약화 파라미터 → 구부정/보상 패턴
- **민감도 분석:** 파라미터 스윕에서 보행 지표의 점진적, 단조적 변화
- **보조기기 상호작용:** 외골격/AFO 공동 시뮬레이션 시 인간 모델의 응답이 임상 관찰과 정성적 일치

### 6.5 검증 지표

| 지표 | 목표 | 비교 데이터 |
|------|------|-----------|
| 관절 토크 RMSE | 최대 토크의 15% 이내 | 보행 연구실 역동역학 |
| 보행 속도 / 활보장 / 분속수 | 표준 ± 1 SD 이내 | 출판 표준치 |
| 지면반력 패턴 | 이중 정점 패턴 재현 | 지면반력계 데이터 |
| 진자 검사 진동 횟수 | ± 1회 이내 일치 | 임상 진자 검사 기록 |
| 관절 임피던스 vs 동시수축 | 단조 증가 | 임피던스 측정 문헌 |
