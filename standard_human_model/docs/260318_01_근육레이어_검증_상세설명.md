## T01 — Hill Active Force-Length (활성 힘-길이 관계)

### 생리적 메커니즘

근섬유는 액틴(actin)과 마이오신(myosin) 필라멘트의 가교 결합(cross-bridge)으로 힘을 발생시킨다.
이 가교 결합 수는 두 필라멘트의 겹침 정도에 따라 달라진다.

- 근육이 너무 짧으면: 두 액틴 필라멘트가 서로 겹쳐서 마이오신이 결합할 자리가 감소 → 힘 감소
- 최적 길이(l_opt)에서: 겹침 면적이 최대 → 가교 결합 수 최대 → 힘 최대
- 근육이 너무 길면: 액틴-마이오신 간격이 벌어져서 결합 불가 → 힘 감소

이 관계를 **활성 힘-길이 커브(Active F-L Curve)**라 하며, 정규화하면 종 모양(bell curve)이 된다.

### 구현 방식

```
f_FL = exp( -((l_norm - 1.0)^2) / (2 * 0.45^2) )
```

가우시안 함수로 근사한다. 중심 = 1.0(최적 길이), 표준편차 = 0.45는 Thelen 2003 기본값이다.
`l_norm = l_muscle / l_opt`로 정규화하므로, l_opt를 기준으로 상대적 길이를 표현한다.

최종 active force 계산에서 이 값이 activation, F_max, f_FV와 곱해진다:
```
F_active = activation * F_max * f_FL(l_norm) * f_FV(v_norm) * cos(pennation)
```

### 테스트 내용

l_norm을 0.3~2.0까지 200개 포인트로 스윕하여 f_FL 값을 계산한다.
피크가 l_norm=1.0 ± 0.01 안에 있고 피크값 ≥ 0.99이면 PASS.

### PASS 의미

가우시안 수식이 올바르게 구현됐고, 정규화 중심이 정확히 1.0(최적 길이)에 있다는 뜻이다.

### 부가 진단 (경고 표시)

현재 `healthy_baseline.yaml`에서 `l_opt=1.0`(무차원)이고 `l_slack=0.30m`이므로
실제 동작점이 `l_norm = 0.30 / 1.0 = 0.30`이 된다.
이 지점에서 f_FL ≈ 0.30 → 최대 힘의 30%만 발생한다.
이는 코드 버그가 아니라 YAML 파라미터 이슈다. 플롯의 빨간 점선이 이 경고를 시각화한다.

## T02 — Hill Force-Velocity (힘-속도 관계)

### 생리적 메커니즘

가교 결합의 힘 발생은 마이오신 헤드의 회전 속도에 의존한다.

- 수축(concentric, v < 0): 근육이 짧아지는 방향. 가교가 회전하면서 힘을 발생시키는데,
  수축 속도가 빠를수록 마이오신이 액틴에 붙어있는 시간이 짧아지고
  각 가교당 생성되는 충격량이 감소한다 → 힘 감소.
  최대 수축 속도(v_max)에 가까워지면 힘 → 0.

- 등척성(isometric, v = 0): 가교가 최적 속도로 순환 → 기준 힘(f_FV = 1.0).

- 신장(eccentric, v > 0): 외부 힘이 근육을 늘리는 방향. 가교가 끌려가면서
  스프링처럼 탄성 에너지가 더해진다 → 등척성보다 큰 힘. 최대 약 1.8배.
  이것이 내리막 보행이나 충격 흡수에서 큰 힘이 발생하는 이유다.

이 관계를 **Hill 방정식**이라 하며 1938년 Archibald V. Hill의 실험에서 유래한다.

### 구현 방식

수축 구간과 신장 구간에 각각 다른 수식을 적용하는 piecewise 함수다.

수축 구간 (v_norm < 0):
```
f_FV = (1 + v_norm) / (1 - v_norm / 0.25)
```
v_norm=-1이면 분자=0 → f_FV=0. v_norm=0이면 f_FV=1.0.

신장 구간 (v_norm > 0):
```
f_FV = 1 + 0.8 * v_norm / (v_norm + 0.18),  clamp max=1.8
```
점근적으로 1.8에 수렴한다.

수치 안정성을 위해 v_norm을 -0.99로 하한 clamp하여 분모=0 방지한다.

### 테스트 내용

v_norm을 -1.0~0.8까지 300개 포인트로 스윕.
f_FV(0)=1.0, f_FV(-0.99)<0.05, max≤1.81 세 조건을 모두 만족하면 PASS.
단조성(수축→감소, 신장→증가)도 시각적으로 확인한다.

### PASS 의미

Hill 방정식의 세 경계조건(등척성=1, 최대수축→0, eccentric max≤1.8)이 모두 맞다.
수식의 분기(concentric/eccentric)와 수치 처리가 정확하다.

## T03 — Hill Passive Force-Length (수동 힘-길이 관계)

### 생리적 메커니즘

활성화 없이도 근육이 충분히 늘어나면 저항력이 발생한다.
이는 두 가지 구조에서 온다.

1. 타이틴(titin) 단백질: 사르코미어 내부의 분자 스프링. 근육이 최적 길이를 넘어 늘어나면
   탄성 저항을 발생시킨다. 이것이 수동 힘의 주된 원천이다.
2. 근막과 결합 조직: 근육을 둘러싼 결합 조직이 추가 저항을 제공한다.

핵심은 **최적 길이 이하에서는 수동 힘이 거의 없고**, 최적 길이를 초과할 때부터 급격히 증가한다.
이 특성 덕분에 관절이 지나치게 과신전될 때 수동적으로 보호된다.

### 구현 방식

```
f_PE = k_pe * max(0, l_norm - 1.0)^2 / epsilon_0
```

`max(0, ...)` 때문에 l_norm ≤ 1.0에서는 정확히 0이 된다.
l_norm이 1.0을 초과하는 순간부터 제곱 증가. k_pe와 epsilon_0는 가파름을 조절하는 파라미터다.

### 테스트 내용

l_norm=1.0에서 passive force가 0인지(< 1e-5) 확인.
l_norm=1.6에서 손계산 값 `4.0 * 0.36 / 0.6 = 2.4`와 코드 출력이 일치(오차 < 0.01)하면 PASS.

### PASS 의미

수동 힘의 임계값(l_norm=1.0)과 수식이 정확히 일치한다.
현재 YAML 파라미터 이슈로 실제 시뮬레이션에서 passive force가 발생하지 않는 것은
코드 버그가 아니라 `l_opt` 파라미터 튜닝 문제임을 이 테스트가 구분해준다.

## T04 — Activation Linearity (활성화 선형성)

### 생리적 메커니즘

운동 신경(motor neuron)이 근섬유를 발화(fire)시키면 활성화(activation)가 발생한다.
더 많은 신경 발화 → 더 많은 근섬유 모집(recruitment) → 활성화 증가.

Active force는 활성화에 **선형 비례**한다:
```
F_active = activation × (F_max × f_FL × f_FV × cos(pennation))
```
즉, activation=0.5는 activation=1.0의 정확히 절반의 active force를 발생시킨다.
이것은 신경 모집 원리의 수학적 표현이다.

단, passive force는 activation과 무관하게 길이에만 의존하므로,
총 힘 = activation × active + passive 구조다.

### 구현 방식

```
F_total = activation * F_max * f_FL * f_FV * cos_penn  +  F_max * f_PE  +  damping * F_max * v
```

activation이 앞의 active 항에만 곱해지므로, passive를 분리하면 active는 선형이다.

### 테스트 내용

activation=0.0, 0.5, 1.0 세 레벨에서 l_norm 전 범위를 스윕.
passive를 제거한 후 `active(0.5) / active(1.0)` 비율을 l_norm 0.7~1.3 구간 평균으로 계산.
0.5 ± 0.02이면 PASS.

### PASS 의미

근육 모집 원리가 정확히 구현됐다.
세 곡선이 세로 방향으로 1:2 비율을 이루면, 정책이 activation 값을 0~1 사이로 출력할 때
물리적으로 의미 있는 힘 스케일링이 일어난다는 것을 보장한다.

## T05 / T05b — Moment Arm 커플링 히트맵 (이관절근 검증)

### 생리적 메커니즘

근육은 하나 또는 두 개의 관절에 걸쳐 있다.

**단관절근(mono-articular)**: 한 관절만 지나감.
예) soleus(발목만), quadriceps 일부(무릎만).

**이관절근(bi-articular)**: 두 관절에 동시 작용.
예)
- hamstrings(슬건근): 고관절 신전 + 무릎 굴곡을 동시에 담당
- gastrocnemius(비복근): 무릎 굴곡 + 발목 족저굴곡을 동시에 담당
- rectus femoris(대퇴직근): 고관절 굴곡 + 무릎 신전을 동시에 담당

이관절근은 에너지 전달 측면에서 중요하다. 예를 들어 보행 중 pushing-off 단계에서
발목과 무릎 파워를 동시에 조절한다. 이 이중 결합이 R 행렬(Moment Arm Matrix)에
두 관절 열 모두 비영 값으로 반영되어야 한다.

**R 행렬의 역할**:
```
l_muscle = l_slack - R @ q        (관절 각도에서 근육 길이 계산)
tau = F_muscle @ R                 (근육 힘에서 관절 토크 계산)
```
R은 (num_muscles × num_dofs) 행렬. 양의 R 값 → 관절이 신전될 때 근육이 짧아짐.

### 구현 방식

`muscle_definitions.yaml`에서 각 근육의 R 행렬을 직접 정의한다.
이관절근은 두 관절 DOF에 비영 값을 가지도록 수작업으로 설정.
OpenSim Rajagopal 2016 모델의 해부학적 측정값을 근거로 한다.

```python
tau = R.forces_to_torques(F)   # F @ R_matrix
```

### 테스트 내용

T05: 20개 근육 각각에 F=100N을 인가하여 8개 주요 관절의 토크를 20×8 히트맵으로 시각화.
노란 테두리 = 이관절근. 색이 있는 셀 개수를 육안으로 확인.

T05b: hamstrings_L/R, gastrocnemius_L/R, rectus_femoris_L/R 6개 근육이
각각 ≥ 2개 관절에 0.1 Nm 이상 토크를 내면 PASS.

### PASS 의미

R 행렬 정의가 해부학적으로 올바르다. 이관절근이 두 관절에 동시 영향을 준다.
보행 시뮬레이션에서 발목-무릎 협응, 고관절-무릎 협응이 물리적으로 가능하다.

## T06 — L/R 대칭 검증

### 생리적 메커니즘

인체는 좌우 대칭이다. 왼쪽 hamstrings와 오른쪽 hamstrings는 동일한 해부학적 구조를 가지며,
동일한 힘을 발생시키면 각자의 관절에 동일한 크기의 토크를 생성해야 한다.

시뮬레이션에서 L/R 대칭이 깨지면 다음과 같은 문제가 발생한다:
- 보행 시 좌우 비대칭 토크로 직선 보행 불가
- 환자 프로필을 적용할 때 편측 마비를 의도치 않게 시뮬레이션함
- RL 학습 시 불필요한 대칭 깨짐으로 학습 불안정

### 구현 방식

`muscle_definitions.yaml`에서 L 근육군과 R 근육군의 R 행렬 값을 동일하게 정의한다.
단, L 근육은 L 관절 DOF에만, R 근육은 R 관절 DOF에만 비영 값을 가진다.
서로 교차(L 근육 → R 관절 등)하면 안 된다.

### 테스트 내용

L/R 대응 근육 쌍(예: hamstrings_L vs hamstrings_R)에 각각 독립적으로 F=100N 인가.
L 관절 토크와 R 관절 토크를 비교하여 오차 < 1e-4 Nm이면 PASS.

### PASS 의미

L/R 교차 오염이 없고 YAML 정의가 정확히 대칭이다.
결과: 최대 오차 = 0.00e+00 Nm (완전 일치).
이는 YAML을 동일 값으로 복사했기 때문에 수치적으로 완전 일치가 보장된다.

## T07 — Ligament Soft-Limit (인대/관절낭 연성 한계)

### 생리적 메커니즘

관절에는 두 가지 한계가 있다.

**Hard limit**: 뼈가 물리적으로 닿는 극단 범위. 시뮬레이터에서 rigid body collision으로 처리.

**Soft limit**: 관절낭(joint capsule)과 인대(ligament)가 제공하는 탄성 저항.
정상 ROM(가동범위) 경계에 가까워지면 인대가 늘어나기 시작하고,
경계를 초과하면 지수적으로 증가하는 저항 토크가 발생한다.
이 토크는 hard limit에 도달하기 전에 관절을 되돌리는 방향으로 작용한다.

환자별 차이:
- 구축(contracture): soft limit이 정상보다 안쪽에서 시작(ROM 감소) + 가파르게 증가
- 이완(flaccid): soft limit 저항이 약화

### 구현 방식

```
τ_ligament = -k_lig * exp(alpha * max(0, q - q_soft_upper))
           + k_lig * exp(alpha * max(0, q_soft_lower - q))
```

상한 초과 시 음의 토크(관절을 줄이는 방향), 하한 미만 시 양의 토크(관절을 키우는 방향).
`k_lig`는 강성 크기, `alpha`는 지수 기울기(가파름)다.

### 테스트 내용

L_Knee DOF에 `soft_lower=-0.1 rad, soft_upper=2.0 rad`를 설정하고
무릎 각도를 -0.5~2.8 rad까지 스윕.

상한(2.0) 초과 구간에서 토크가 전부 음수인지,
중립 구간(0.5~1.8 rad)에서 |tau| < 0.1 Nm인지 확인. 둘 다 만족하면 PASS.

### PASS 의미

인대 복원 토크의 방향(부호)과 활성화 구간이 정확하다.
이 모델이 없으면 보행 중 관절이 정상 ROM을 벗어나도 저항 없이 과신전/과굴곡이 발생한다.

## T08 — Stretch Reflex (신장 반사): Healthy vs Spastic

### 생리적 메커니즘

신장 반사(stretch reflex)는 척수 수준의 피드백 루프다.

**정상 메커니즘**:
근방추(muscle spindle)가 근육 신장 속도를 감지한다.
→ Ia 구심성 신경이 척수로 신호를 전달한다.
→ 척수에서 α 운동 신경을 직접 활성화한다.
→ 같은 근육이 반사적으로 수축하여 신장에 저항한다.

이 반사는 갑작스러운 외란에서 관절을 보호하고 자세를 유지한다.

**경직(spasticity, UMN 손상)**:
뇌졸중, 뇌성마비 등 상위운동신경원(UMN) 손상 시 척수 내 억제 회로가 손상된다.
결과: stretch reflex gain이 과도하게 증가, 더 낮은 속도에서도 반사가 발동된다.
임상적으로 velocity-dependent resistance(속도의존 저항)로 나타나며
수동적인 관절 운동 시 과도한 저항감이 생긴다.

**이완(flaccidity, LMN 손상)**:
하위운동신경원(LMN) 손상 시 α 운동 신경이 직접 손상된다.
결과: 반사가 완전히 소실되고 근육이 힘을 내지 못한다.

### 구현 방식

```
a_stretch = max(0, v_muscle - threshold) * stretch_gain
a_total = clamp(descending_cmd + a_stretch, 0, 1)
```

환자 유형별 파라미터:
- Healthy: `stretch_gain=1.0`, `stretch_threshold=0.1 (l_opt/s)`
- Spastic: `stretch_gain=8.0`, `stretch_threshold=0.02 (l_opt/s)`
  → 낮은 속도에서도 반사 발동, 8배 강한 활성화
- Flaccid: `stretch_gain=0.0` → 반사 완전 소실

### 테스트 내용

근육 신장 속도를 0~2.0(l_opt/s)까지 스윕.
v=0.5에서 Spastic > Healthy인지, v=0에서 둘 다 ≈ 0인지 확인.

### PASS 의미

임상적으로 알려진 경직과 정상의 차이(더 낮은 역치, 더 강한 반응)가
수치로 재현됐다.
환자 시뮬레이션에서 `stretch_gain` 하나로 UMN/LMN 차이를 표현할 수 있다는 것이 검증됐다.

## T09 — Full Pipeline 제로 입력 (전체 파이프라인 조립 검증)

### 생리적 메커니즘

근육이 쉬고 있을 때(cmd=0, vel=0)를 생각해보자.
해부학적 중립 자세에서는 어떤 토크도 발생하지 않아야 한다.

- descending command = 0 → 운동 신경 발화 없음 → active force = 0
- 속도 = 0 → stretch reflex 발동 없음 → reflex activation = 0
- ROM 중간 자세 → 인대가 늘어나지 않음 → ligament torque = 0
- 최적 길이 부근 → passive force 최소

즉 "아무것도 안 할 때 토크가 없어야 한다"는 것은 파이프라인 전체의 정합성을 검증하는 기준이다.

### 중요한 SMPL 좌표계 이슈

SMPL 골격에서 `q=0`은 무릎 완전 신전(하한선)이다. 해부학적 중립이 아니다.
무릎 ROM은 [0°, 145°]이므로, q=0은 경계값이다.
인대 모델이 이 경계에서 복원 토크를 발생시키는 것은 정상 동작이다.
이것이 시나리오 A에서 큰 토크가 나와도 버그가 아닌 이유다.

진짜 "중립"은 각 관절 ROM의 중간값 `(lower + upper) / 2`다. 이것이 시나리오 B다.

### 구현 방식

전체 파이프라인 흐름:
```
compute_torques(dof_pos, dof_vel, cmd, dt)
  ↓ l_muscle = l_slack - R @ q         (MomentArmMatrix)
  ↓ v_muscle = -(R @ dq)               (MomentArmMatrix)
  ↓ a_cmd = reflex(cmd, v_muscle, F)   (ReflexController)
  ↓ activation = activation_dynamics.step(a_cmd, dt)
  ↓ F_muscle = hill_model(activation, l, v)
  ↓ tau_muscle = F_muscle @ R
  ↓ tau_ligament = ligament.compute_torque(dof_pos, dof_vel)
  ↓ tau_total = tau_muscle + tau_ligament
```

### 테스트 내용

시나리오 A: q=0 (진단 플롯만, FAIL 판정 없음)
  → 어느 관절에서 인대 토크가 발생하는지 확인하는 설계 이해용

시나리오 B: q=ROM center, cmd=0, vel=0
  → max|tau| < 5.0 Nm이면 PASS

### PASS 의미

파이프라인 6단계를 조립했을 때 각 서브모듈이 정상적으로 연결됐다.
T01~T08이 각 모듈을 독립적으로 검증했고, T09는 이들이 하나로 연결됐을 때
예상치 못한 상호작용이 없다는 것을 확인한다.
결과: max|tau| = 0.000 Nm. 수식 전체가 일관됐다는 수치적 증거다.

## 수학 모델 출처 및 타 시뮬레이터와의 비교

### 출처

각 모듈의 수식은 근골격계 시뮬레이션 분야의 표준 참고문헌에 기반한다.

| 모듈 | 출처 |
|------|------|
| Active F-L (가우시안, width=0.45) | Thelen 2003 (J. Biomech. Eng.) |
| F-V (Hill 방정식, concentric/eccentric 분기) | Thelen 2003, Hill 1938 원저 |
| Passive F-L (이차 함수) | Millard 2013 (J. Biomech. Eng.) |
| Moment Arm R 행렬 값 | OpenSim Rajagopal 2016 전신 모델에서 근사 |
| Activation dynamics (1차 ODE) | Thelen 2003 |
| Stretch reflex (gain/threshold) | 임상 근전도 문헌 (경직: gain 5~15, threshold 0.01) |

모두 수십 년간 실험적으로 검증된 모델이다.

### OpenSim과의 비교

OpenSim은 `Thelen2003Muscle` 또는 `Millard2012EquilibriumMuscle` 클래스를 제공하며,
F-L/F-V 수식의 수학적 형태가 우리 구현과 동일한 계보다.
단, 다음 두 가지에서 구현 충실도 차이가 있다.

첫째, R 행렬. OpenSim은 관절 각도의 함수로 `R(q)`를 실시간 계산한다.
무릎 굴곡 각도에 따라 hamstrings의 moment arm이 20~30% 변한다.
우리 구현은 상수 R을 사용하므로, 자세에 따른 moment arm 변화가 반영되지 않는다.

둘째, 건(tendon) 모델. OpenSim은 건의 탄성을 equilibrium solver로 풀지만,
우리는 rigid tendon 근사(건 길이 고정)를 사용한다.

### MyoSuite와의 비교

MyoSuite(Meta AI)는 MuJoCo 기반으로 Hill 모델 레이어를 추가한 구조다.
F-L/F-V 수식은 De Groote 2016에 기반하며, Thelen과 유사하지만
연속 미분 가능하도록 smooth approximation을 적용한 버전이다.
R 행렬은 MuJoCo MJCF에서 실시간 Jacobian으로 자동 계산된다.

### 우리 구현의 현재 위치

| 항목 | OpenSim/MyoSuite | 우리 구현 | 영향 |
|------|-----------------|-----------|------|
| F-L/F-V 수식 계보 | Thelen/Millard/De Groote | **동일 (Thelen 2003)** | — |
| R 행렬 | R(q), 각도 의존 | **상수 R** | 자세별 moment arm 오차 |
| Passive F-L 함수 | 지수 함수 (Millard 정확) | 이차 함수 (근사) | passive force 형태 약간 상이 |
| Tendon | equilibrium solver | rigid tendon 근사 | 건 탄성 무시 |
| l_opt 단위 | meters | **현재 1.0 (버그)** | 근육이 30% 동작점에서 작동 |

핵심 수식은 동일한 문헌 계보에서 왔고 T01~T09로 코드 구현이 검증됐다.
R(q) 각도 의존성은 중장기 개선 과제다.

## 알려진 파라미터 이슈 (코드 버그 아님)

현재 `healthy_baseline.yaml`의 `l_opt: 1.0`은 무차원 값으로 설정되어 있다.
`l_slack`은 미터 단위(0.30m)이므로 `l_norm = 0.30 / 1.0 = 0.30`이 된다.
이 지점에서 active force가 최대의 약 30%에 불과하고 passive force는 발생하지 않는다.

수정 방향: `l_opt`를 실제 최적 근섬유 길이(미터)로 설정한다.
예시: quadriceps 0.08m, hamstrings 0.10m, gastrocnemius 0.05m, soleus 0.04m.
이 수정 후 T01 플롯의 빨간 점선이 1.0 근처로 이동하고 passive force도 발생하기 시작할 것이다.
