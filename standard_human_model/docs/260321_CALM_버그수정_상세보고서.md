## BUG-01 — l_opt 단위 버그 (Critical)

### 문제 진단

파이프라인 전체의 단위 불일치가 연쇄적으로 영향을 미치고 있었다.

```
moment_arm.py                    muscle_model.py (수정 전)
───────────────                  ──────────────────────
l_slack = 0.25~0.35 (m)         l_opt = 1.0 (무차원)
R @ q ≈ 0.02 (m)
                                 l_norm = muscle_length / l_opt
muscle_length                           = 0.30 / 1.0
  = l_slack - R@q                       = 0.30   ← 최적 길이의 30%!
  ≈ 0.30 (m)
                                 f_FL(0.30) ≈ 0.25  ← 가우시안에서 거의 바닥
```

근육 능동력이 최대의 약 25%밖에 나오지 않는 심각한 버그.
추가로 v_norm과 damping에도 단위 불일치가 있었다.

### 수정 내용

#### (1) `config/healthy_baseline.yaml` — 파라미터 실측값 교체

Rajagopal 2016 Table 2 기준. 전체 20개 근육 (L+R) 수정.

```yaml
# 수정 전                          수정 후
l_opt: 1.0          →              l_opt: 0.044           # m (soleus 예시)
l_tendon_slack: 1.0  →              l_tendon_slack: 0.260   # m
```

전체 근육 l_opt 값:

| 근육군 | l_opt (m) | l_tendon_slack (m) | 출처 |
|--------|-----------|-------------------|------|
| hip_flexors | 0.100 | 0.133 | iliopsoas 대표 |
| gluteus_max | 0.142 | 0.125 | |
| hip_abductors | 0.054 | 0.065 | glut_med 대표 |
| hip_adductors | 0.138 | 0.110 | add_long 대표 |
| quadriceps | 0.099 | 0.136 | vasti 대표 |
| rectus_femoris | 0.076 | 0.346 | bi-articular |
| hamstrings | 0.109 | 0.326 | bicep_fem_lh 대표 |
| gastrocnemius | 0.051 | 0.408 | medial head |
| soleus | 0.044 | 0.260 | |
| tibialis_ant | 0.068 | 0.223 | |

#### (2) `config/muscle_definitions.yaml` — l_slack 재계산

l_slack은 관절 중립(0도)에서의 MTU 전체 길이. 다음 공식으로 재계산:

```
l_slack = l_opt * cos(pennation) + l_tendon_slack
```

예시 (soleus):
```
l_slack = 0.044 * cos(0.44) + 0.260
        = 0.044 * 0.905 + 0.260
        = 0.300 m
```

이 값이 moment_arm.py에서 `compute_muscle_length()`의 기준점이 된다.

#### (3) `core/muscle_model.py` — 섬유 길이/속도 계산 전면 수정

**수정 전 (잘못된 코드):**
```python
l_norm = muscle_length / self.l_opt            # MTU 길이를 직접 나눔 (단위 불일치)
v_norm = muscle_velocity / self.v_max          # 단위 불일치
F_damping = self.damping * self.f_max * muscle_velocity  # raw m/s 사용
```

**수정 후:**
```python
# Rigid tendon 가정: 건 길이 불변 → 섬유가 전체 길이 변화를 흡수
cos_penn = torch.cos(self.pennation)

# 1. 섬유 길이 (fiber length)
l_ce = (muscle_length - self.l_tendon_slack) / cos_penn
l_ce = torch.clamp(l_ce, min=1e-4)  # 음수 방지
l_norm = l_ce / self.l_opt

# 2. 섬유 속도 (fiber velocity)
v_ce = muscle_velocity / cos_penn
v_norm = v_ce / (self.v_max * self.l_opt)  # v_max는 l_opt/s → 절대속도 = v_max * l_opt

# 3. Damping도 정규화 속도 사용
F_damping = self.damping * self.f_max * v_norm
```

**수정 후 데이터 플로우 (soleus 예시):**
```
moment_arm.py:
  l_mtu = l_slack - R@q = 0.300 - (-0.05 * q_ankle) ≈ 0.300 m (중립)

muscle_model.py:
  l_ce = (0.300 - 0.260) / cos(0.44) = 0.040 / 0.905 = 0.044 m
  l_norm = 0.044 / 0.044 = 1.0  ← 최적 길이! f_FL(1.0) = 1.0

  v_mtu ≈ 0.03 m/s (보행 중 전형적)
  v_ce = 0.03 / 0.905 = 0.033 m/s
  v_norm = 0.033 / (6.0 * 0.044) = 0.033 / 0.264 = 0.125  ← 합리적 범위
```

## BUG-02 — R 행렬 각도 의존성 없음 (High)

### 문제 진단

Moment arm은 관절 각도에 따라 변화한다. 예: hamstrings의 무릎 moment arm은 0도에서 3.5cm, 90도에서 5.5cm. 상수 R로는 이 변화를 포착할 수 없어 관절 극단에서 토크 오차 최대 30%.

### 수정 내용

`core/moment_arm.py`에 2차 다항식 moment arm 추가.

#### 구현 구조

```python
# 모듈 레벨 상수: polynomial 계수 딕셔너리
_POLY_COEFFS = {
    (muscle_name, joint_name, axis_idx): (a0, a1, a2),
    ...
}

# 클래스 내부: 텐서화된 polynomial 계산
class MomentArmMatrix:
    def _compute_R(self, dof_pos):
        """
        polynomial 계수가 있는 항목만 동적 계산.
        나머지는 R_const (상수) 유지.
        """
        R = self.R_const.expand(num_envs, ...).clone()
        q = dof_pos[:, poly_dof_indices]
        r_poly = a0 + a1*q + a2*q^2
        R[:, muscle_idx, dof_idx] = r_poly
        return R  # (num_envs, num_muscles, num_dofs)
```

#### 적용된 polynomial 계수 (18개 항목)

| 근육 | 관절 | a0 | a1 | a2 | 비고 |
|------|------|-----|-----|-----|------|
| hamstrings | Hip | -0.060 | -0.008 | +0.002 | 굴곡 시 moment arm 감소 |
| hamstrings | Knee | +0.030 | +0.015 | -0.005 | 굴곡 시 moment arm 증가 |
| quadriceps | Knee | -0.040 | -0.012 | +0.004 | patella 전방 이동 효과 |
| rectus_femoris | Knee | -0.040 | -0.012 | +0.004 | quadriceps와 동일 |
| gastrocnemius | Knee | +0.020 | +0.008 | -0.003 | |
| gastrocnemius | Ankle | -0.050 | +0.005 | +0.002 | 배굴 시 감소 |
| soleus | Ankle | -0.050 | +0.005 | +0.002 | gastrocnemius와 유사 |
| tibialis_ant | Ankle | +0.030 | -0.003 | -0.001 | |

L/R 대칭이므로 실제 18개 = 9쌍.

#### API 변경사항

```python
# 수정 전
tau = R.forces_to_torques(F_muscle)

# 수정 후 (dof_pos 인자 추가)
tau = R.forces_to_torques(F_muscle, dof_pos)
```

`compute_muscle_length(dof_pos)`와 `compute_muscle_velocity(dof_pos, dof_vel)`는 기존에도 dof_pos를 받고 있었으므로 API 변경 없음. 내부적으로 상수 R 대신 `_compute_R(dof_pos)`를 사용하도록 변경.

polynomial이 없는 경우 (hip_flexors, gluteus_max 등)는 기존과 동일하게 상수 R 사용.

#### 성능 고려

- polynomial이 없으면 `_compute_R()`가 `R_const`를 그대로 반환 (2D 텐서) → 기존과 동일한 행렬곱
- polynomial이 있으면 3D 텐서 `(num_envs, num_muscles, num_dofs)` 반환 → `torch.bmm` 사용
- polynomial 계산 자체는 벡터화되어 있으므로 오버헤드 미미

## BUG-03 — Elastic Tendon 미구현 (High)

### 문제 진단

아킬레스건(soleus, gastrocnemius)의 tendon/fiber 비율이 5~9로 매우 크다.
보행 에너지의 35~40%가 건의 탄성 에너지 저장·반환에서 온다.
rigid tendon 가정은 이를 완전히 무시하여 push-off 파워를 과소평가한다.

### 수정 내용

`core/muscle_model.py`에 elastic tendon 옵션 추가.

#### Newton-Raphson 섬유 길이 수렴

핵심 아이디어: rigid tendon에서는 `l_ce = (l_mtu - l_ts) / cos(penn)`으로 해석해가 바로 나오지만,
elastic tendon에서는 건도 늘어나므로 `F_tendon = F_muscle` 평형을 반복법으로 풀어야 한다.

```python
def _compute_fiber_state_elastic(self, activation, muscle_length, muscle_velocity):
    cos_penn = torch.cos(self.pennation)

    # 초기 추정: rigid tendon 해
    l_ce = (muscle_length - self.l_tendon_slack) / cos_penn

    for _ in range(max_iter):   # max_iter = 10
        # 건 길이 = MTU - 섬유*cos(penn)
        l_tendon = muscle_length - l_ce * cos_penn
        l_tendon_norm = l_tendon / self.l_tendon_slack

        # 건 힘 (De Groote 2016 simplified)
        f_t = k_tendon * max(0, l_tendon_norm - 1.0)^2

        # 근육 힘: activation * f_FL * cos(penn) + f_PE
        f_m = activation * f_FL(l_ce/l_opt) * cos_penn + f_PE(l_ce/l_opt)

        # 잔차
        residual = f_t - f_m
        if |residual| < 1e-3: break

        # Jacobian (해석적 근사)
        df_t/dl_ce = -cos_penn / l_ts * 2 * k_tendon * strain
        df_m/dl_ce = (수치 미분, dl=1e-5)

        # Newton step
        l_ce -= residual / (df_t - df_m)
```

#### 사용법

```python
# soleus = index 8, gastrocnemius = index 7 (L), 17, 18 (R) 등
model = HillMuscleModel(
    num_muscles=20,
    num_envs=512,
    elastic_tendon_indices=[7, 8, 17, 18],  # gastroc_L, soleus_L, gastroc_R, soleus_R
)
```

elastic tendon이 지정되지 않은 근육은 기존과 동일하게 rigid tendon으로 계산.
혼합 모드: rigid 전체 계산 후 elastic 근육만 덮어쓰기.

#### 건 힘 모델

De Groote 2016 simplified quadratic:
```
f_T = k_tendon * max(0, l_tendon/l_ts - 1.0)^2
```
k_tendon = 35 (기본값). strain이 0이면 힘 = 0, strain이 클수록 급격히 증가.

## BUG-04 — Stretch Reflex 지연 없음 (Medium)

### 문제 진단

두 가지 버그가 있었다:

#### (a) 기본값 문제

```python
# 수정 전
reflex_delay_steps: int = 1
```

dt = 1/120s ≈ 8.3ms. delay_steps=1 → 8ms 지연.
실제 척수 반사弓 지연은 25~60ms (평균 ~40ms).

#### (b) Circular buffer off-by-one 버그

```python
# 수정 전: buffer 크기 = delay_steps
self._delay_buffer = torch.zeros(reflex_delay_steps, ...)  # 크기 1

# _apply_delay():
self._delay_buffer[self._buffer_idx] = signal       # slot 0에 쓰기
delayed_idx = (self._buffer_idx + 1) % 1  = 0       # slot 0 읽기 ← 방금 쓴 것!
```

delay_steps=1일 때 buffer 크기가 1이므로 write slot과 read slot이 동일.
결과: 실질 지연 = 0. 반사 신호가 즉각 반영됨.

### 수정 내용

#### (a) 기본값 변경

```python
# 수정 후
reflex_delay_steps: int = 5    # 5 * 8.3ms ≈ 41.7ms (척수 반사弓 지연 범위 내)
dt: float = 1.0 / 120.0        # 정보 기록용
```

#### (b) Buffer 로직 수정

```python
# 수정 후
buf_size = max(reflex_delay_steps + 1, 1)   # +1로 write/read 겹침 방지
self._delay_buffer = torch.zeros(buf_size, num_envs, num_muscles, ...)

def _apply_delay(self, signal):
    # 현재 slot에 쓰기
    self._delay_buffer[self._buffer_idx] = signal

    # delay_steps만큼 과거 slot 읽기
    delayed_idx = (self._buffer_idx - self.reflex_delay_steps) % self._buf_size
    delayed_signal = self._delay_buffer[delayed_idx].clone()

    # 인덱스 전진
    self._buffer_idx = (self._buffer_idx + 1) % self._buf_size
    return delayed_signal
```

**검증 (delay_steps=5, buf_size=6):**
```
step 0: write[0], read[(0-5)%6]=read[1] → 초기값 0 (정상: 아직 5스텝 전 데이터 없음)
step 1: write[1], read[(1-5)%6]=read[2] → 0
...
step 5: write[5], read[(5-5)%6]=read[0] → step 0의 signal ← 정확히 5스텝 지연!
step 6: write[0], read[(6-5)%6]=read[1] → step 1의 signal ← 정확히 5스텝 지연!
```

## 추가 발견/수정 사항

### Damping 단위 버그 (BUG-01 과정에서 발견)

```python
# 수정 전: raw MTU 속도(m/s) 직접 사용
F_damping = self.damping * self.f_max * muscle_velocity
# → soleus: 0.05 * 2800 * 0.03 = 4.2 N
# → tibialis: 0.05 * 600 * 0.03 = 0.9 N
# 같은 MTU 속도라도 근육마다 다른 v_max 무시 → 비물리적

# 수정 후: 정규화된 섬유 속도 사용
F_damping = self.damping * self.f_max * v_norm
# v_norm = v_ce / (v_max * l_opt) → 근육별 최대 속도 대비 비율
# → 물리적으로 일관된 감쇠력
```

### forces_to_torques API 변경

```python
# 수정 전
tau = moment_arm.forces_to_torques(F_muscle)

# 수정 후 (polynomial R 사용 시 dof_pos 필요)
tau = moment_arm.forces_to_torques(F_muscle, dof_pos)
# dof_pos=None 이면 상수 R 사용 (하위 호환)
```
