## 1. 검증 목표 재정의

> **최종 목표**: 이 시뮬레이션이 실제 인간의 보행을 믿을 만하게 재현하여,
> **키네마틱(joint angles)**, **GRF(지면반력)**, **EMG(근활성화 타이밍)**
> 세 가지 데이터를 동시에 실제와 유사하게 생성할 수 있음을 검증한다.

이 목표를 달성하려면 다음 질문들이 순서대로 "YES"여야 한다:

```
L0: 수학식이 맞나?
  ↓ YES
L1: 모듈들이 올바르게 연결되나?
  ↓ YES
L2: 물리 엔진과 연동이 정확한가?
  ↓ YES
L3: 알려진 생체역학적 현상이 재현되나?
  ↓ YES
L4: 실제 보행 데이터(Kin+GRF+EMG)와 얼마나 가까운가?
  ↓ YES
L5: 환자군별 병리 패턴이 재현되나?
```

---

## 2. 검증 계층 구조 (5-Level Pyramid)

```
          ▲
         /L5\      환자 특이적 병리 보행 재현
        /────\
       / L4   \     실제 데이터 정량 비교 (Kin + GRF + EMG)
      /────────\
     /   L3     \   현상론적 타당성 (알려진 생체역학 현상)
    /────────────\
   /     L2       \ 물리 엔진 연동 정확성
  /────────────────\
 /       L1         \ 모듈 간 데이터 흐름
/────────────────────\
          L0           수학 단위 테스트
```

아래로 내려갈수록 **기반**이고, 위로 올라갈수록 **최종 목표**에 가깝다.
하위 레벨이 PASS되지 않으면 상위 레벨 결과를 신뢰할 수 없다.

### Level 0 — 수학 단위 테스트 (Unit Mathematics)

**질문**: "개별 수식이 수학적으로 올바른가?"

**검증 항목**:

| 테스트 | 검증 내용 | 합격 기준 |
|--------|---------|---------|
| L0-01 | Hill F-L active curve | 피크 l_norm=1.0 ± 0.01, 최대값 ≥ 0.99 |
| L0-02 | Hill F-V curve | FV(0)=1.0, 단조 감소 (concentric), 1.5배 이하 (eccentric) |
| L0-03 | Hill F-L passive | l_norm ≤ 1.0에서 0, l_norm > 1.0에서 단조 증가 |
| L0-04 | 활성화 × 힘 선형성 | F(a=0.5) ≈ 0.5 × F(a=1.0) (±5%) |
| L0-05 | 인대 soft-limit 방향성 | ROM 초과 시 복원 토크 방향 정확 |
| L0-06 | Stretch reflex gain 효과 | gain=8 > gain=1 > gain=0 |
| L0-07 | 좌우 대칭 | 동일 조건에서 L/R 토크 크기 오차 < 1e-4 Nm |
| L0-08 | 영 입력 → 영 출력 | u=0, q=중립, v=0 → max\|τ\| < 5 Nm |
| L0-09 | 활성화 ODE 시정수 | tau_act 측정값이 설정값 ± 20% |

### Level 1 — 모듈 데이터 흐름 (Module Integration)

**질문**: "모듈들이 올바른 순서로, 올바른 차원으로 연결되는가?"

이 레벨은 물리 엔진 없이 순수 PyTorch만으로 검증한다.
파이프라인 각 단계의 **텐서 shape, 값 범위, 수치 안정성**을 확인한다.

**현재 커버**: 05_deep_validation 일부 (V5-5, V5-6) ← 분산되어 있음

**구현할 테스트 목록** (`validation/01_module_flow/` 신설):

| 테스트 | 파일 | 검증 내용 |
|--------|------|---------|
| L1-01 | `test_tensor_shapes.py` | `compute_torques()` 입출력 shape 일치성 확인. 모든 중간 텐서 (muscle_length, activation, F_muscle, tau_muscle) shape 로그 출력 |
| L1-02 | `test_moment_arm_coupling.py` | R 행렬에서 bi-articular 근육(hamstrings, gastrocnemius, rectus_femoris)이 실제로 2개 이상 관절에 값 가지는지 확인. coupling 강도 정량화 |
| L1-03 | `test_activation_pipeline.py` | `u → reflex → activation_dyn → a` 단계별 값 범위가 [0, 1]인지, 시정수 비대칭(act < deact)이 유지되는지 |
| L1-04 | `test_muscle_velocity_sign.py` | 무릎 굴곡 방향 각속도 → hamstrings 속도 음수(단축) / quadriceps 속도 양수(신장). 부호 규약 명세와 일치 확인 |
| L1-05 | `test_force_torque_mapping.py` | 단일 근육 힘 F → R^T × F = τ. 알려진 R 값에서 수기 계산과 일치 (절대 오차 < 1e-6 Nm) |
| L1-06 | `test_numerical_stability.py` | 극단 입력(u=1.0 전체 활성화, v_max 속도, ROM 한계)에서 NaN/Inf 없음. 1000 step 연속 실행에서 발산 없음 |
| L1-07 | `test_reset_consistency.py` | reset() 후 activation=0 확인. 다른 env_ids 리셋이 나머지 환경에 영향 없음 |

**핵심 테스트: L1-04 (속도 부호)**

근육 속도 부호는 `compute_torques()` 파이프라인에서 가장 혼동하기 쉬운 부분이다.
현재 `moment_arm.py`의 `compute_muscle_velocity = -(R @ dq)` 규약이 생리학 정의와 맞는지 확인:

```python
# 검증: 무릎 굴곡 (q_knee 증가) 시
# hamstrings: 단축(shortening) → v_muscle < 0 (힘 증가, F-V 기준)
# quadriceps: 신장(lengthening) → v_muscle > 0 (힘 감소)
# 이 부호가 FV 함수에 올바르게 들어가는지 추적
```

### Level 3 — 현상론적 타당성 (Phenomenological Validation)

**질문**: "알려진 생체역학적 현상이 재현되는가?"

수식과 연동이 맞아도, 전체 시스템 동작이 생물학적으로 말이 되는지는 별개다.
이 레벨은 문헌에서 잘 알려진 질적 현상들이 관찰되는지 확인한다.

**현재 커버**: `04_comprehensive_validation/`, `05_deep_validation/` 일부

**체계적으로 정리할 테스트 목록** (`validation/03_phenomena/` 신설):

| 테스트 | 현상 | 기대 결과 | 문헌 근거 |
|--------|------|---------|---------|
| L3-01 | **경직(Spasticity) 클로누스** | stretch_gain 임계값(~3.0) 이상에서 지속 진동 발생 | Lance 1980 |
| L3-02 | **Ankle push-off 파워 버스트** | 보행 말기 입각기(terminal stance)에서 발목 토크 급증 | Neptune 2001 |
| L3-03 | **공동수축(Co-contraction)** | 관절 불안정 상황에서 주동/길항근 동시 활성화 | Winter 1984 |
| L3-04 | **Stretch reflex 지연** | 신전 자극 후 반사 활성화까지 delay_steps × dt 지연 확인 | Sherrington 1910 |
| L3-05 | **GTO 활성 억제** | F_muscle이 임계값 초과 → 길항근 억제 (reciprocal) | Houk 1967 |
| L3-06 | **수동-능동 힘 전환점** | l_norm~1.3 이상에서 수동 힘이 능동 힘 추월 | Thelen 2003 |
| L3-07 | **Bi-articular 에너지 전달** | gastrocnemius가 무릎에서 흡수 → 발목으로 전달 | Bobbert 1986 |

**L3-07 상세 (Bi-articular 에너지 전달)**은 CALM의 핵심 주장 중 하나이므로 별도 실험으로 정의:


### Level 5 — 환자 특이적 병리 패턴 (Patient-Specific Pathology)

**질문**: "환자 파라미터를 바꾸면 알려진 병리 보행이 재현되는가?"

| 테스트 | 환자군 | 기대 병리 패턴 | 정량 기준 |
|--------|-------|-------------|---------|
| L5-01 | **편마비 (Stroke)** | 환측 무릎 신전 구축 (Stiff Knee Gait): 유각기 무릎 굴곡 감소 | 환측 swing peak knee flex < 35° (정상 60°) |
| L5-02 | **편마비** | 발끌기 방지 circumduction: 고관절 외전 보상 | 환측 hip ABduction > 5° during swing |
| L5-03 | **파킨슨** | 보폭 감소, 보행 속도 감소 | step_length < 0.7 × healthy |
| L5-04 | **파킨슨** | 발끌기 (shuffling): swing 최고점 < 2 cm | foot clearance < 2 cm |
| L5-05 | **SCI 완전** | 수동 유각 (무활성화): EMG ON 구간 없음 | 근활성화 합산 < 5% 정상 |

## 8. "믿을 만한 시뮬레이션"의 정량적 기준

최종 목표를 달성했다고 판단하는 기준을 수치로 정의한다.

### 최소 기준 (Minimum Viable)

| 채널 | 기준 |
|------|------|
| 키네마틱 | 무릎/고관절/발목 각도 패턴이 정성적으로 보행과 유사 (쌍봉 패턴 등) |
| GRF | 수직 GRF 쌍봉 패턴 재현, 피크 0.8~1.5 BW 범위 |
| EMG | 6개 근육의 ON 구간이 Perry 1992 구간과 ±20% 이내 |

→ 이 수준이면 **논문 A 제출** 가능

### 목표 기준 (Target)

| 채널 | 기준 |
|------|------|
| 키네마틱 | RMSE < 8° (hip/knee/ankle 각각) vs Winter normative |
| GRF | RMSE < 0.15 BW 전 구간 |
| EMG | 타이밍 오차 < ±10% gait cycle, 6개 근육 |

→ 이 수준이면 **논문 B (임상 번역)** 기여 가능

### 이상적 기준 (Ideal, 장기)

| 채널 | 기준 |
|------|------|
| 키네마틱 | 특정 환자 측정 데이터에서 파라미터 규명 → 동일 환자 재현 |
| GRF | 환자별 비대칭 패턴 재현 (편마비: 환측 하중 감소) |
| EMG | 환자별 근활성화 패턴 재현 (뇌졸중: co-contraction 증가) |

→ 이 수준이면 **논문 C (end-to-end 임상 번역)** 가능
