## 핵심 목표

> 시뮬레이션이 실제 인간 보행의 **키네마틱(joint angles)**, **GRF(지면반력)**, **EMG(근활성화 타이밍)**
> 세 채널을 동시에 믿을 만하게 재현함을 증명한다.

```
             ▲  신뢰도 높아짐

           /────\
          / L4   \     실제 데이터 정량 비교  ← 최우선 미구현
```

## Level별 상태 및 테스트 목록

### L0 — 수학 단위 테스트 ✅ 완료 (9/9)

**질문**: 개별 수식이 수학적으로 맞는가?
**도구**: PyTorch만, IsaacGym 불필요
**폴더**: `validation/01_muscle_layer/`

| ID | 검증 내용 | 합격 기준 |
|----|---------|---------|
| L0-01 | Hill F-L active curve 형태 | 피크 l_norm=1.0 ± 0.01 |
| L0-02 | Hill F-V curve 단조성 | FV(0)=1.0, concentric 단조 감소 |
| L0-03 | Hill F-L passive curve | l_norm ≤ 1.0에서 0, 이후 단조 증가 |
| L0-04 | 활성화-힘 선형성 | F(a=0.5) ≈ 0.5 × F(a=1.0) ±5% |
| L0-05 | 인대 soft-limit 방향성 | ROM 초과 시 복원 방향 정확 |
| L0-06 | Stretch reflex gain 효과 | gain 8 > gain 1 > gain 0 |
| L0-07 | 좌우 대칭 | L/R 동일 조건 → 토크 오차 < 1e-4 Nm |
| L0-08 | 영 입력 → 영 출력 | u=0, q=중립 → max\|τ\| < 5 Nm |
| L0-09 | 활성화 시정수 측정값 | 측정 tau_act가 설정값 ±20% |

### L1 — 모듈 데이터 흐름 ⬜ 미구현 (0/7)

**질문**: 모듈들이 올바른 차원과 부호로 연결되는가?
**도구**: PyTorch만, IsaacGym 불필요
**폴더**: `validation/L1_module_flow/` (신설)

| ID | 검증 내용 | 핵심 확인 |
|----|---------|---------|
| L1-01 | 텐서 shape 검사 | 파이프라인 전 단계의 (num_envs, num_muscles) shape 일치 |
| L1-02 | Moment arm 커플링 | hamstrings/gastrocnemius/rectus_femoris가 R 행렬에서 ≥2 관절에 비영 값 |
| L1-03 | 활성화 파이프라인 범위 | u→reflex→activation 모든 단계에서 값 ∈ [0, 1] |
| L1-04 | **근육 속도 부호** | 무릎 굴곡(dq>0) → hamstrings 단축(v<0), quadriceps 신장(v>0) |
| L1-05 | 힘-토크 변환 수치 | 단일 근육 F×R^T = τ 수기 계산 일치, 오차 < 1e-6 Nm |
| L1-06 | 수치 안정성 | 극단 입력(u=1.0 전체, v=v_max) 1000 step에서 NaN/Inf 없음 |
| L1-07 | reset 일관성 | env_ids 일부 리셋이 나머지 환경 activation에 영향 없음 |

**L1-04가 가장 중요**: `moment_arm.py`의 `-(R @ dq)` 부호 규약이 `muscle_model.py`의 FV 함수 입력 방향과 맞는지 추적하는 테스트.

**폴더**: `validation/L3_phenomena/` (04+05 재편)

| ID | 현상 | 기대 결과 | 문헌 |
|----|------|---------|------|
| L3-01 | 경직 클로누스 onset | stretch_gain > 임계값에서 지속 진동 | Lance 1980 |
| L3-02 | Ankle push-off 파워 버스트 | 말기 입각기에서 발목 토크 급증 | Neptune 2001 |
| L3-03 | 공동수축(Co-contraction) | 불안정 조건에서 주동/길항근 동시 활성화 | Winter 1984 |

### L4 — 데이터 정량 비교 ⬜ 미구현 (0/4) ← 핵심

**질문**: 실제 보행 데이터(Kin+GRF+EMG)와 얼마나 가까운가?
**도구**: IsaacGym + reference_data/ YAML
**폴더**: `validation/L4_data_fidelity/` (신설)

**참조 데이터** (별도 YAML 파일로 관리):
- `reference_data/winter2009_normative.yaml` — 관절 각도 + GRF (Winter 2009)
- `reference_data/perry1992_emg_timing.yaml` — EMG 타이밍 (Perry 1992)

| ID | 검증 내용 | 합격 기준 (최소) | 합격 기준 (목표) |
|----|---------|--------------|--------------|
| L4-01 | 키네마틱 vs Winter 2009 | hip/knee/ankle ROM 오차 < ±15° | RMSE < 8° 각 관절 |
| L4-02 | GRF vs Winter 2009 | 수직 GRF 쌍봉 패턴 재현, 피크 0.8~1.5 BW | RMSE < 0.15 BW 전 구간 |
| L4-03 | EMG 타이밍 vs Perry 1992 | 6개 근육 ON 구간 ±20% GC 이내 | 타이밍 오차 < ±10% GC |
| L4-04 | 종합 Fidelity Score | score > 0.5 | score > 0.7 |

**종합 점수 계산**:
```
score_kin  = 1 - RMSE(sim, ref_angle) / ROM_ref
score_grf  = 1 - RMSE(sim_grf, ref_grf) / peak_ref
score_emg  = 1 - mean_timing_error / 15%
total_score = 0.4×score_kin + 0.3×score_grf + 0.3×score_emg
```

**"믿을 만한 시뮬레이션" 기준**:

| 수준 | 기준 | 의미 |
|------|------|------|
| 최소 (Minimum) | 쌍봉 GRF + ROM 정성 유사 + EMG ±20% | 논문 A 제출 가능 |
| 목표 (Target) | RMSE < 8°, GRF < 0.15 BW, EMG ±10% | 논문 B 기여 가능 |
| 이상 (Ideal) | 환자별 측정 데이터로 파라미터 규명 후 재현 | 논문 C 가능 |

**폴더**: `validation/L5_patient_pathology/` (신설)

| ID | 환자군 | 기대 병리 | 정량 기준 |
|----|-------|---------|---------|
| L5-01 | Stroke | Stiff Knee Gait (swing 굴곡 감소) | 환측 swing peak knee flex < 35° |
| L5-02 | Stroke | Circumduction (고관절 외전 보상) | 환측 hip abduction > 5° during swing |
| L5-03 | Parkinson | 보폭 감소 | step_length < 0.7 × healthy |
| L5-04 | Parkinson | 발끌기 | foot clearance < 2 cm |
| L5-05 | SCI | 자발적 근활성화 소실 | EMG 합산 < 5% 정상 |
| L5-06 | 공통 | 손상 심각도 ↑ → 기능 지표 단조 감소 | mild < severe 모든 지표 방향 일치 |
