### L1-04: Muscle Velocity Sign Convention ✅ PASS (CRITICAL)

가장 중요한 테스트. `moment_arm.py`의 `-(R @ dq)` 부호가 `muscle_model.py`의 FV 함수와 일치하는지 확인.

| 동작 | 근육 | 속도 | 해석 | 정상? |
|------|------|------|------|-------|
| Knee flexion (dq>0) | hamstrings | -0.030 | 단축 | ✅ |
| Knee flexion (dq>0) | quadriceps | +0.040 | 신장 | ✅ |
| Knee flexion (dq>0) | gastrocnemius | -0.020 | 단축 | ✅ |
| Knee flexion (dq>0) | rectus_femoris | +0.040 | 신장 | ✅ |
| Hip flexion (dq>0) | hip_flexors | -0.040 | 단축 | ✅ |
| Hip flexion (dq>0) | gluteus_max | +0.060 | 신장 | ✅ |
| Ankle dorsiflexion (dq>0) | soleus | +0.050 | 신장 | ✅ |
| Ankle dorsiflexion (dq>0) | tibialis_ant | -0.030 | 단축 | ✅ |

부호 규약: `v_mtu = -(R @ dq)`. 양수 = MTU 신장(늘어남), 음수 = MTU 단축(짧아짐).
이 규약은 FV curve 입력과 일치: v_norm > 0 → eccentric (f_FV > 1.0), v_norm < 0 → concentric (f_FV < 1.0).

### L1-05: Force-Torque Conversion ✅ PASS

수기 계산과 코드 출력 비교. 오차 0.

| 조건 | 수기 계산 | 코드 출력 | 오차 |
|------|---------|---------|------|
| hamstrings 1000N → hip torque (q=0) | -60.0 Nm | -60.0000 Nm | 0 |
| hamstrings 1000N → knee torque (q=0) | 30.0 Nm | 30.0000 Nm | 0 |
| hamstrings 1000N → knee torque (q=0.5rad) | 36.25 Nm | 36.2500 Nm | 0 |

Polynomial R(q) 검증: 무릎 0.5rad (28.6도)에서 hamstrings moment arm이 0.030 → 0.03625로 증가.
R(0.5) = 0.030 + 0.015×0.5 + (-0.005)×0.25 = 0.03625. 코드 출력과 정확히 일치.

### L1-06: Numerical Stability ✅ PASS

극단 입력으로 1000 step 시뮬레이션.

| 항목 | 결과 |
|------|------|
| NaN 발생 | 0회 |
| Inf 발생 | 0회 |
| max |torque| | ~2.6e23 Nm |

max torque가 매우 큰 이유: dof_pos ~ N(0, 2.0 rad) → 일부 환경에서 ROM을 수 라디안 초과 → ligament 지수 함수가 매우 큰 복원 토크 생성. 이는 물리적으로 정상 (비현실적 자세에서는 당연히 큰 복원력).

**발견 및 수정**: 초기 실행에서 Inf 2회 발생. 원인은 `ligament_model.py`의 `exp(alpha * excess)`에서 excess가 너무 클 때 overflow. `exp` 인자를 `max=50.0`으로 clamp하여 해결 (exp(50) ≈ 5.2e21, 충분히 큰 복원력).
