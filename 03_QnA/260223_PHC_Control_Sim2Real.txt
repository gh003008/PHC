1. 제어 아키텍처 (Control Architecture)
----------------------------------------
- 정책(Policy)의 역할: 강화학습 모델은 각 관절이 가야 할 목표 각도(Target Position)를 결정합니다. 이를 기구학적 명령Action이라고 부릅니다.
- 하위 제어기(Low-level Controller): AI가 내린 각도 명령을 실제 토크(Torque)로 변환하는 역할입니다. PHC 시뮬레이션에서는 환경 코드(humanoid.py) 내의 PD 제어 로직이 이 역할을 수행합니다.

2. 시뮬레이션상의 PD 제어 구현
----------------------------------------
- 위치: humanoid.py 파일의 _compute_torques 함수 내에 구현되어 있습니다.
- 공식: 토크 = P_gain * (목표각도 - 현재각도) - D_gain * 현재속도
- 특징: 로봇 모델(XML/URDF)에 정의된 강성 값을 사용하지 않고, 환경 코드에서 직접 토크를 계산하여 물리 엔진에 주입하는 방식을 사용합니다. 이는 실제 로봇의 모터 드라이버 특성을 더 세밀하게 묘사하기 위함입니다.

3. Sim2Real 관점에서의 의미
----------------------------------------
- 실제 로봇에서는 이 PD 계산 과정을 모터 드라이버(Firmware)가 전담하게 됩니다.
- 시뮬레이션 환경 자체에서 이 과정을 소프트웨어로 구현함으로써 실제 로봇과 시뮬레이션 사이의 간극을 줄이고 하드웨어 제어기를 더 정확하게 모사할 수 있습니다.

4. VIC (Variable Impedance Control) 확장
----------------------------------------
- 일반 모델: Action 크기가 69이며, 관절의 강성(P_gain)이 고정된 상태에서 명령을 내립니다.
- VIC 모델: Action 크기가 138로 늘어나며, AI가 목표 각도뿐만 아니라 관절의 강성(P_gain, D_gain)까지 실시간으로 조절하는 지능을 배웁니다.
- 적용 효과: VIC를 사용하면 지면 상태나 동작의 속도에 따라 관절을 유연하게 혹은 단단하게 조절하여 더 인간다운 보행이 가능해집니다.


PD 제어의 계산 sim 2 real 
1. 전담 주체 및 계산 위치 (어디서 계산하나?)
시물레이션 환경에서는 "하위 제어기(Motor Driver)"의 역할을 Python 코드(humanoid.py)가 전담합니다.
실제 로봇: 
Policy (PC/Jetson) -> Target Angle (CAN/EtherCAT) 
-> Motor Driver (Firmware PD) -> Motor Torque

PHC 시뮬레이션: 
Policy (Neural Net) -> Target Angle (Action) 
-> humanoid.py (Python PD) -> PhysX (Force Input)

2. 모델 정의 단계의 P, D 값 vs 코드상의 P, D 값
모델 정의(XML/URDF) 단계에서도 stiffness(P), damping(D) 값을 설정할 수 있지만, 
PHC는 이를 무시하고 코드에서 직접 제어하는 방식을 택하고 있습니다.

XML/URDF 값: Isaac Gym의 내장 제어기(Internal PD)를 쓸 때 사용되는 값입니다.
PHC 방식: 내장 제어기를 쓰지 않고, set_dof_actuation_force_tensor라는 함수를 통해 
"내가 직접 계산한 토크를 강제로 주입하겠다"고 선언합니다.
이유: Sim2Real을 할 때 실제 모터 드라이버의 특성을 더 정확하게 시뮬레이션 코드에 반영하고, 
나중에 배울 VIC(강성 가변 제어)처럼 $K_p, K_d$ 값을 실시간으로 바꾸기 위해서입니다.
