1. 시각화 도구 비교 (Open3D vs Isaac Gym)
----------------------------------------
- Isaac Gym Viewer (기본 창): 수백 개의 에이전트가 동시에 학습하는 모습을 격자 형태로 보여주는 메인 화면입니다. 전체적인 학습 양상을 파악하기에 좋으나 개별 에이전트의 자세를 정밀하게 보기는 어렵습니다.
- Open3D 렌더러 (--render_o3d): 특정 1명의 에이전트와 레퍼런스 모션을 사람 모양의 메시(Mesh) 형태로 겹쳐서 보여주는 특수 렌더러입니다. 관절의 꺾임이나 자세의 일치도를 분석할 때 매우 유용합니다.

2. 시각화 영상 녹화 방법
----------------------------------------
- Open3D 창: 시각화 창이 활성화된 상태에서 키보드의 'L' 키를 누르면 녹화가 시작되고, 다시 누르면 종료됩니다. 결과물은 output/renderings/ 폴더에 mp4 파일로 자동 저장됩니다.
- Isaac Gym 창: 코드 내부에 직접적인 녹화 기능은 없으므로, OBS Studio나 SimpleScreenRecorder와 같은 외부 화면 녹화 프로그램을 권장합니다.

3. 학습 곡선 (Reward Curve) 분석
----------------------------------------
- 계단식 상승 (Staircase): 강화학습 에이전트가 특정 물리적 요령(예: 무릎 펴기, 무게 중심 이동)을 깨닫는 순간 보상이 급격히 점프하는 현상이 발생합니다. 이는 매우 정상적인 학습 과정입니다.
- 보상 값의 진동: RSI(Random State Initialization) 기법으로 인해 에이전트가 매번 다른 시점에서 시작하므로 평균 보상은 항상 어느 정도 흔들리게 됩니다.
- 보상 하락 현상: AMP 학습의 특성상 에이전트를 감별하는 판별기(Discriminator)가 강해지면 상대적으로 보상이 낮아 보일 수 있으나, 이는 실제로는 에이전트가 더 정교한 동작을 배우는 과정입니다.

4. 에피소드 종료 및 루핑 조건
----------------------------------------
- 사이클 모션 (cycle_motion): 기본적으로 True로 설정되어 있어 레퍼런스 모션이 짧더라도 처음으로 돌아가 무한 반복하며 걷습니다.
- 조기 종료 (Early Termination): 에피소드가 10초를 채우지 못하고 끝나는 이유는 모션이 끝나서가 아니라, 에이전트가 넘어졌기 때문입니다. (골반 높이가 0.15m 이하로 떨어지면 실패로 간주하고 리셋)
