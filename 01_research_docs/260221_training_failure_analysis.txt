# PHC 학습 실패 분석 및 복구 전략 통합 리포트

## 1. 개요
본 문서는 PHC(Perceptual Control Hierarchy) 모델 및 VIC(Variable Impedance Control) 구현 과정에서 발생한 주요 학습 실패 사례와 원인을 분석하고, 향후 동일한 실수를 반복하지 않기 위한 복구 전략을 정리합니다.

## 2. 주요 실패 사례 분석

### 사례 1: PNN 체크포인트 호환성 실패 (차원 불일치)
*   **현상**: 기존 학습된 PNN 체크포인트(574차원)를 현재 환경(454차원)에 적용하기 위해 0-패딩(Zero-padding)을 사용했으나, 캐릭터가 즉시 넘어짐 (Episode Length ~2, Reward ~8).
*   **원인**: 단순히 입력 차원 수만 맞추는 것으로는 부족함. PNN은 입력값의 물리적 의미와 순서(Topology)에 결합되어 있는데, 환경이 달라지면서 관측값 매핑이 깨짐.
*   **교훈**: 원본 체크포인트와 환경의 관측값 구성이 다를 경우, 억지로 맞추기보다 처음부터 학습(Scratch)하거나 Fine-tuning을 수행해야 함.

### 사례 2: 복합 시계열 구조(MCP + PNN)의 Scratch 학습 실패
*   **현상**: Composer와 4개의 PNN을 동시에 처음부터 학습(`fitting: False`, `models: []`)시켰으나, 20시간 이상의 학습 후에도 성능 향상이 전혀 없었음.
*   **원인**: 
    1. **구조적 복잡성**: 상위 Composer가 랜덤하게 초기화된 4개의 하위 PNN을 조합하여 학습하는 방식은 초기 탐색 공간이 지나치게 넓어 수렴이 매우 어려움.
    2. **그래디언트 전달 문제**: RL-Games 알고리즘이 복잡한 계층 구조를 거쳐 PNN의 세부 가중치까지 효율적으로 최적화하지 못함.
*   **교훈**: 계층적 구조(MCP)를 학습할 때는 'Bottom-up' 접근법을 따라야 함. 단일 동작(Primitive)들을 먼저 개별적으로 학습시켜 고정시킨 후, 상위 조합기(Composer)를 학습시키는 것이 정석임.

## 3. 백투베이직(Back-to-Basics) 전략

현재의 복잡한 설정을 모두 걷어내고, 가장 기초적인 단계부터 다시 시작하여 가용성을 확보합니다.

### 단계 1: 단일 프리미티브(Single Primitive) 학습
*   **Task**: `HumanoidIm` (MCP/PNN 구조가 아닌 단일 MLP 정책 사용)
*   **목표**: '서기(Standing)' 동작 모방 성공 및 안정적인 Baseline 확보.
*   **설정**: `isaac_pd` 제어 모드, `num_envs` 512(리소스 안정성), `amass_isaac_standing_upright_slim.pkl` 모션 사용.

### 단계 2: 안정된 동작 기반 확장
*   단계 1이 성공하여 캐릭터가 중심을 잡으면, 이를 바탕으로 걷기(Walking) 동작으로 확장하거나 PNN 구조로 전이 학습 시도.

## 4. 결론
복잡성(MCP)과 호환성(Old Checkpoints) 문제를 동시에 해결하려다 발생한 시행착오를 바탕으로, 현재는 "작고 확실한 성공"을 목표로 하여 단일 동작부터 차근차근 Baseline을 구축하고 있습니다.
